### 准备一：硬件兼容性自检（300字）

#### 为什么要检查？
- 避免"装完发现显卡/声卡/网卡不能用"的尴尬

#### 如何检查？
1. 查看官方硬件兼容性列表（HCL）
2. 使用Live USB试用模式测试
3. 搜索自己硬件型号 + Ubuntu 26.04 关键词

#### 重点检查项
- 显卡（NVIDIA/AMD/Intel）
- 无线网卡（尤其是Realtek芯片）
- 蓝牙模块
- 触控板（笔记本用户）

---

### 准备二：数据备份方案（400字）

#### 必备备份清单

| 备份内容 | 备份方式 | 推荐工具 |
|:---|:---|:---|
| 个人文档 | 云同步/外置硬盘 | Nextcloud/rclone |
| 浏览器数据 | 账号同步 | Firefox Sync/Chrome Sync |
| 开发环境配置 | 配置文件导出 | dotfiles（.bashrc, .vimrc等） |
| 虚拟机/容器 | 导出镜像 | Docker save/VirtualBox导出 |

#### 备份验证
- 备份完成后，验证关键文件可正常打开
- 建议保留两份备份（云端+本地）

---

### 准备三：分区规划策略（400字）

#### 常见分区方案对比

| 方案 | 适用场景 | 分区设置 |
|:---|:---|:---|
| 简单方案 | 新手/硬盘空间小 | 仅分 / 和 swap |
| 推荐方案 | 日常使用 | / (50GB) + /home (剩余) + swap (8GB) |
| 进阶方案 | 服务器/高级用户 | / (30GB) + /home + /var (20GB) + swap |

#### 分区避坑指南
- EFI分区必须存在（UEFI启动），建议512MB
- /home单独分区：重装系统不丢失个人数据
- swap大小：与内存相当或略小（SSD时代）

---

### 准备四：启动盘制作要点（300字）

#### 工具选择
- Rufus（Windows，推荐）
- balenaEtcher（macOS/Linux）
- Ventoy（支持多系统）

#### 关键设置
- GPT分区方案（UEFI启动）
- 关闭Secure Boot（部分主板需要）
- 验证ISO校验和（避免镜像损坏）

### 准备五：安装后必做清单预览（200字）

#### 安装完成后第一步要做什么？
- 更新系统：sudo apt update && sudo apt upgrade
- 安装常用软件：输入法、浏览器、编辑器
- 配置开发环境（开发者）

#### 提前准备
- 列出自己常用的软件清单
- 准备好配置文件备份
- 记录网络配置信息（静态IP等）
