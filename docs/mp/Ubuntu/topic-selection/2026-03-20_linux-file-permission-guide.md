### 第一部分：权限的本质——三个角色，三种权利（600字）

#### 三个角色

| 角色 | 英文 | 含义 | 典型场景 |
|:---:|:---:|:---|:---|
| 所有者 | Owner (u) | 文件创建者 | 你的个人文档 |
| 所属组 | Group (g) | 文件所属的用户组 | 项目组共享文件 |
| 其他人 | Others (o) | 其他所有用户 | 系统全局配置 |

#### 三种权利

| 权限 | 符号 | 对文件 | 对目录 |
|:---:|:---:|:---|:---|
| 读 | r (read) | 查看文件内容 | 列出目录内容 (ls) |
| 写 | w (write) | 修改文件内容 | 创建/删除目录中的文件 |
| 执行 | x (execute) | 运行程序/脚本 | 进入目录 (cd) |

#### 理解"执行"权限的两个场景
- 对文件：能否运行这个程序/脚本
- 对目录：能否进入这个目录（这是很多人混淆的点）

### 第二部分：读懂权限表示——数字和符号（500字）

#### 符号表示法：rwxr-xr-x

分解示例：drwxr-xr-x
- 第1位：d（目录）/ -（文件）/ l（链接）
- 第2-4位：所有者权限（rwx = 读写执行）
- 第5-7位：所属组权限（r-x = 读执行，无写）
- 第8-10位：其他人权限（r-x = 读执行，无写）

#### 数字表示法：755

| 权限 | 数字计算 | 含义 |
|:---:|:---|:---|
| rwx | 4+2+1 = 7 | 读写执行 |
| rw- | 4+2+0 = 6 | 读写 |
| r-x | 4+0+1 = 5 | 读执行 |
| r-- | 4+0+0 = 4 | 只读 |

#### 常见权限组合速查

| 权限 | 数字 | 典型用途 |
|:---|:---:|:---|
| rwxr-xr-x | 755 | 可执行程序、目录 |
| rw-r--r-- | 644 | 普通文件 |
| rw-rw-r-- | 664 | 团队共享文件 |
| rwx------ | 700 | 私密脚本/目录 |

### 第三部分：修改权限——chmod命令实战（600字）

#### 符号模式（推荐新手）

```bash
# 给所有者添加执行权限
chmod u+x script.sh

# 给所有人添加读权限
chmod a+r file.txt

# 移除其他人的写权限
chmod o-w config.conf

# 批量设置
chmod u=rwx,g=rx,o=r file.txt
```

#### 数字模式（简洁高效）

```bash
# 设置755权限
chmod 755 script.sh

# 递归设置目录权限
chmod -R 755 /var/www/html

# 批量设置文件644、目录755
find . -type f -exec chmod 644 {} \;
find . -type d -exec chmod 755 {} \;
```

#### 实战案例：解决"Permission denied"

```bash
# 场景1：执行脚本提示权限不足
./install.sh
# Permission denied

# 解决：添加执行权限
chmod +x install.sh
./install.sh

# 场景2：编辑系统配置文件
vim /etc/hosts
# Warning: Changing a readonly file

# 解决：使用sudo获取权限
sudo vim /etc/hosts
```

### 第四部分：修改所有者——chown和chgrp（300字）

#### chown：修改所有者

```bash
# 修改文件所有者
sudo chown user1 file.txt

# 同时修改所有者和所属组
sudo chown user1:group1 file.txt

# 递归修改目录
sudo chown -R www-data:www-data /var/www/html
```

#### chgrp：修改所属组

```bash
# 修改文件所属组
sudo chgrp developers file.txt

# 递归修改
sudo chgrp -R developers /opt/project
```

### 第五部分：特殊权限——进阶知识（250字）

#### SUID（4）
- 让程序以所有者身份运行
- 典型案例：/usr/bin/passwd
- 设置：chmod 4755 file

#### SGID（2）
- 目录中新建文件继承所属组
- 团队共享目录常用
- 设置：chmod 2775 /shared

#### Sticky Bit（1）
- 只有文件所有者可删除
- 典型案例：/tmp目录
- 设置：chmod 1777 /tmp

### 附录：常见问题FAQ

#### Q1: 为什么我可以用sudo执行任何操作？
A: sudo让你以root身份运行命令，root拥有最高权限，不受文件权限限制。

#### Q2: 777权限安全吗？
A: 不安全！777表示所有人都可以读写执行，一般只用于临时测试，正式环境应避免。

#### Q3: 文件权限和目录权限有什么区别？
A: 文件权限控制对文件内容的操作，目录权限控制对目录内文件的操作。比如，对目录有写权限，才能在其中创建或删除文件。
