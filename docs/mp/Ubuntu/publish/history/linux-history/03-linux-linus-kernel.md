# Linux 演进史（三）：21岁芬兰学生的一封邮件，如何改变世界？

> **摘要**：1991年，买不起UNIX工作站的芬兰学生林纳斯·托瓦兹，为了一个好用的终端，开始编写一个"业余爱好"的操作系统。他放弃商业限制、拥抱GPL，用务实的态度战胜完美主义，两年内将Linux打造成完整系统。

**1991年8月25日，芬兰赫尔辛基大学。一个21岁的计算机系学生在新闻组发了一封不起眼的邮件：**

> "我正在编写一个（免费的）操作系统（只是个业余爱好，不会像 GNU 那样庞大和专业）……"

**当时没人想到，这个"业余爱好"会成为统治全球服务器、超级计算机和数十亿智能手机的数字基础设施。**

林纳斯·托瓦兹（Linus Torvalds）那年刚满21岁。他有一个所有大学生都有的烦恼：**穷**。

他想要一台能运行 UNIX 的工作站，但商业 UNIX 的授权费要数千美元——这对一个靠打零工买电脑的学生来说，简直是天价。

## 被逼出来的天才：当穷学生遇上"残废"系统

林纳斯东拼西凑，终于买了一台 33MHz 的 386 电脑。这在当时已经是不错的配置了，但他发现了一个尴尬的问题：**没有好用的操作系统**。

商业 UNIX 买不起，他只能安装一个叫 MINIX 的教学用系统。这是荷兰教授安德鲁·塔能鲍姆（Andrew Tanenbaum）为了教学编写的 UNIX 变体，代码只有一万多行，非常适合学生学习操作系统原理。

**但对学生来说很好的东西，对极客来说就是折磨。**

MINIX 就像一辆装了限速器的法拉利——功能被严格限制，很多想做的事情都做不了。最让林纳斯抓狂的是终端仿真器，慢得像蜗牛，功能还残缺。

**"既然买不起好的，那我就自己写一个。"**

这就是 Linux 诞生的起点——不是为了改变世界，只是想有个好用的终端。

## 那封改变世界的邮件：不完美的勇气

1991年8月25日，林纳斯在 comp.os.minix 新闻组发了那封著名的邮件。他说自己在做一个"业余爱好"的操作系统，"不会像 GNU 那样庞大和专业"。

**这封邮件之所以伟大，不是因为它宣告了一个完美软件的诞生，而是因为它展现了一种完全不同的态度：不追求完美，先分享出来。**

在当时的程序员圈子里，大家都有个共识：代码要写到完美无瑕才能公开。但林纳斯在 9 月份就把只有一万行、连编译都会报错的 0.01 版扔到了 FTP 服务器上。

**更关键的是，他做了一个改变历史的决定：放弃最初"禁止商业用途"的限制，全面拥抱斯托曼的 GPL 协议。**

这意味着任何人都可以自由使用、修改、分发这个操作系统。这个决定，让 Linux 从一个学生的"玩具"，变成了全人类的"基础设施"。

## 从玩具到神器：务实的胜利

Linux 的成功，验证了一个商业世界反复证明的道理：**先上线的烂产品，胜过永远不上线的完美产品。**

当时，斯托曼的 GNU 计划已经准备好了所有工具——编译器、编辑器、调试器……只缺一个操作系统内核。但他们的 Hurd 内核因为追求"理论完美"，陷入了无休止的开发泥潭。

而林纳斯的 Linux 呢？代码只有一万行，架构还是被学术派鄙视的"宏内核"。学术泰斗们说这是"过时的设计"。

**但它能跑。而且跑得很快。**

成千上万个买不起 UNIX 的程序员，看到了一个真正能用的系统。他们开始疯狂贡献代码：网卡驱动、声卡支持、内存管理……

**两年时间，这个"业余爱好"变成了完整操作系统。**
