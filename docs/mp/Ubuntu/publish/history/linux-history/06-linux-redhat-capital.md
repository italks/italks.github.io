# Linux 演进史（六）：资本入局与巨头的恐惧——红帽（Red Hat）崛起与“万圣节文件”

从商业文明的发展逻辑来看，一项纯粹发端于民间技术极客的伟大发明，如果在演进的过程中无法寻找到正向的经济循环，它最终难逃萎缩和夭折的宿命。在整个 20 世纪 90 年代，不论 Linux 的“集市模式”多么充满活力，不论它所展现出的社区自治能力多么令人惊叹，一个致命的问题始终盘旋在所有观察者的头顶：

**一个本质上完全免费、甚至连所有源代码都要强制公开的操作系统，究竟要如何赚钱？**

如果没有商业资本的推动去支撑起一套包含售前、测试、培训以及售后响应在内的工业级服务体系，那些手眼通天的跨国巨头（如银行、政府和跨国企业）是绝对不敢将身家性命托付给一个“毫无责任主体”的草台班子操作系统的。

在千禧年钟声即将敲响的前夕，一位名叫鲍勃·杨（Bob Young）的商人与一位名叫马克·尤因（Marc Ewing）的极客，给出了这个世纪难题的最优解。

## “软件免费，服务收费”的商业模式启蒙

在 1993 年创立之初，红帽（Red Hat）公司并不像今天这样是一个庞大的企业级开源巨无霸。马克·尤因，一个习惯于在卡内基梅隆大学戴着康奈尔曲棍球队红色软呢帽到处游荡的技术疯子，自己打包了一个包含了良好安装界面的 Linux 发行版，并以“Red Hat Linux”的名字烧录成光盘。而鲍勃·杨，则是一个拥有极其敏锐嗅觉的计算机推销员。

当他们两人联手后，红帽公司做出了一个极其反直觉的商业决策。

在当时的软件业界，即便是拥抱 Linux 的发行商，也往往试图在附带的专有软件或者纸质说明书上赚取高额利润。然而，红帽却反其道而行之：他们不仅极其严格地遵守了 GPL 协议中的每一项条款，全盘开源；而且对于系统的每一次重大更新，都完全不设置任何技术门槛或版权陷阱。

红帽之所以被称为伟大的企业，在于它在人类软件史上开创了第一套极其稳健的开源变现模式：**“不卖水，只卖水杯和售后服务”**。

本质上，红帽明白，在 Linux 这个日夜不停运转的超级代码集市里，其更新速度以及随之而来的兼容性问题和微小瑕疵是海量的。普通的家庭黑客可以忍受昨天网卡能用今天却突然掉线，但运行千万美元业务流水的大型企业无法忍受。因此，企业真正需要的不是“免费的代码本身”，而是“确定性与责任底线”。红帽将最稳定版本的 Linux 打包，并为企业客户提供极为严苛的 SLA（服务级别协议）：只要你系统宕机了，在我们这买了服务的客户，我就让全球顶尖的工程师打飞的去给你修好，或者给你长达数年的安全补丁保障。

凭借这种颠覆行业认知的商业模式，1999 年 8 月 11 日，红帽公司在纳斯达克成功挂牌上市。在“.com 泡沫”最疯狂的那几年里，它首日股价飙升，甚至成为了华尔街第八大涨幅最猛的科技股。这顶红色软呢帽不仅证明了开源可以长出金子，更在纳斯达克敲响了一记让传统软件帝国胆寒的丧钟。

## 百亿赌局：IBM 的倒戈与微软的震怒

在红帽上市的次年，IT 界的“蓝色巨人”IBM 作出了一个震惊科技界的决定。不仅宣布将在公司内部全面拥抱并采用 Linux，更在随后的 2000 年底放出了一颗重磅炸弹：IBM 将豪掷 10 亿美元（这是一个在当时令人窒息的天文数字）投资 Linux 的研发体系、服务器测试以及开源社区建设。

从历史的博弈角度分析，IBM 绝非仅仅出于对开源精神的浪漫怜悯。在千禧年前后的服务器市场争夺战中，IBM 的专有系统正遭受着同行的惨烈价格战，同时由于缺乏统一的标准，开发生态也日益封闭。投资 Linux，不仅可以依靠其零版税的成本优势打击诸如 Sun Microsystems 这样的硬件宿敌，更能通过抢占开源生态的制高点，在即将爆发的互联网机房基建狂潮中获得压倒性的话语权。更重要的是，这不仅是 IBM 在向昔日的盟友微软公开亮剑宣战，更是彻底为 Linux 这个原本出身草根的操作系统打上了“企业级安全性”的黄金钢印。

至此，Linux 的性质已经完全转变。它不再仅仅是一群无政府主义黑客的玩具俱乐部，而是开始在那些冰冷、肃杀却又极其高可用的大型数据中心服务器集群中，取代包括 UNIX 本尊在内的各种昂贵巨兽，悄悄铺设起了属于下一代互联网数字基座的底层版图。

而这场剧变，终于让在桌面端不可一世的微软陷入了前所未有的恐慌。

## “万圣节文件”引发的核讹诈危机

这极大的触怒了当时的微软帝国掌舵人比尔·盖茨。而在随后的几年里，他的继任者史蒂夫·鲍尔默更是公然将 Linux 骂作“一种从知识产权角度来看如毒瘤一般的存在”。

1998 年的万圣节前夕，一件震惊开源界和科技媒体的丑闻爆发了。开源运动的喉舌埃里克·雷蒙德收到了一份匿名人士泄露的微软内部机密备忘录——这段历史后来被称为“万圣节文件（Halloween Documents）”。

在这些高度机密的内部报告中，微软的工程师坦白承认，开源软件如 Linux 的确存在着长期的系统工程先进性，其代码质量甚至比微软自己的商业软件更加稳定。备忘录显示，开源模式虽然无法在市场营销上直接挑战微软，但它正以一种去中心化的、令微软极其恐惧的方式在服务器领域不断蚕食份额；更糟的是，开源协议中传染性极强的 GPL 以及免费授权模式，“极其严重地威胁了微软依靠卖授权吃饭的根本商业逻辑”。

泄露的文件不仅揭示了微软内部的恐惧，更毫不避讳地探讨了扼杀 Linux 的暗黑策略。微软计划利用其近乎无限的法务及研发资源，通过“拥抱、扩展再消灭（Embrace, Extend, and Extinguish）”的手段——通过在开源标准上强行塞入不兼容的专有扩展协议，或是利用庞大的专利库去起诉 Linux 使用者，以此来在法律层面恐吓企业放弃拥抱开源系统。

这种在暗处策动的“核讹诈”，意味着属于 Linux 最致命、最血腥的至暗考验即将到来。不仅是微软，所有在旧时代享受着知识产权垄断暴利的大公司都在蠢蠢欲动，试图从法律的绞肉机里寻求对这个年轻操作系统的最后一击。

而引发这场世纪绞杀战的导火索，正捏在一家名叫 SCO 的流氓公司手中。
