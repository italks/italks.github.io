# Linux 演进史（六）：微软最恐惧的敌人，竟然不靠卖软件赚钱

> **摘要**：1998年微软泄露的"万圣节文件"承认Linux代码质量更优。红帽公司用"软件免费、服务收费"模式上市即受追捧，IBM豪赌10亿美元投资Linux。开源终于找到了可持续的商业路径。

**1998年，微软内部泄露了一份机密文件。**

工程师们在报告中承认：Linux 的代码质量比 Windows 更稳定，开源模式正在威胁微软的商业模式。

**他们建议的策略是："拥抱、扩展、消灭"。**

这就是著名的"万圣节文件"——微软对 Linux 的恐惧，暴露无遗。

而就在同一年，一家叫红帽的公司，给出了一个让微软更害怕的答案：**免费的东西，也能赚钱。**

## 红帽的颠覆性商业模式

1999年8月11日，红帽在纳斯达克上市。**股价首日飙升，成为华尔街第八大涨幅最猛的科技股。**

红帽的商业模式很简单：**软件免费，服务收费。**

企业不关心代码是否免费，他们关心的是：**系统崩溃时，有人负责吗？**

红帽给企业客户提供了 SLA（服务级别协议）：系统崩溃，我们派全球顶尖工程师去修；提供数年安全补丁保障。

**这就是"卖水杯"而不是"卖水"的智慧。**

## IBM 的豪赌：10亿美元投给"免费"系统

2000年，IBM 宣布投资10亿美元发展 Linux。

**这让微软彻底慌了。**

泄露的"万圣节文件"显示，微软承认 Linux 代码质量更优，正在威胁其商业模式。他们制定了"拥抱、扩展、消灭"的策略。

**但一切都太晚了。Linux 已经成为企业级的标准。**

## 为什么 IBM 敢押注"免费"系统？

很多人不理解：IBM 为什么要投资一个"免费"的操作系统？

答案是：**IBM 不靠卖操作系统赚钱。**

IBM 真正的业务是：卖服务器硬件、卖咨询服务、卖企业解决方案。对他们来说，操作系统只是一个载体。

过去，IBM 每卖一台服务器，就要给微软交一份 Windows 授权费。这相当于每赚一笔钱，都要给竞争对手交"过路费"。

而 Linux 呢？免费。更重要的是，IBM 可以参与 Linux 的开发，让它更好地适配自己的硬件和服务。

**这就是开源的商业逻辑：让基础设施免费，在更高价值的领域赚钱。**

后来，亚马逊用同样的逻辑做 AWS，谷歌用同样的逻辑做 Android。

## 红帽模式的成功密码

红帽的成功，揭示了开源商业化的真正密码：

**1. 企业客户不怕花钱，怕的是"没人负责"。**

当服务器在凌晨三点崩溃时，CIO 需要的是一个能打电话叫来工程师的电话号码，而不是"请去 Stack Overflow 上提问"的回复。

**2. "免费"降低了试用门槛，"服务"创造了持续收入。**

企业可以先免费试用 Linux，确认好用后再购买服务。这比"先付几十万授权费再决定好不好用"的模式友好得多。

**3. 开源让客户"锁定"变得更难，但也让供应商更有动力持续改进。**

红帽知道，客户随时可以切换到其他 Linux 发行版。这倒逼红帽必须持续提供高质量的服务，而不是像传统软件公司那样"卖完就不管了"。

## 微软的误判：从恐惧到拥抱

微软的"万圣节文件"提出了"拥抱、扩展、消灭"的策略：
1. **拥抱**：表面上支持开源，开发兼容版本
2. **扩展**：添加私有功能，让开源版本变得"不完整"
3. **消灭**：利用私有功能锁定用户，最终消灭竞争对手

这个策略在浏览器战争中成功过，IE 就是这么打败 Netscape 的。

但微软低估了一件事：**开源社区的学习能力。**

当他们试图对 Linux 使用同样的策略时，发现根本无法"锁定"用户——代码是公开的，任何私有扩展都可以被社区复制。

最终，微软不得不改变策略。2014年，新任 CEO 纳德拉宣布"微软爱 Linux"。2016年，微软成为 Linux 基金会的白金会员。

**敌人变成了盟友——这就是开源的力量。**
