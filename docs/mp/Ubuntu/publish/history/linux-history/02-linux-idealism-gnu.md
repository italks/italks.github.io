# Linux 演进史（二）：理想主义的反叛——斯托曼、GNU 计划与 GPL 协议

从历史的角度看，任何伟大的变革在最初往往是由少数不妥协的理想主义者发起的。在 20 世纪 80 年代，当整个软件产业都沉浸在闭源与商业化的狂欢中时，第一代黑客赖以生存的空气正在变得极其稀薄。“代码封闭”与“版权协议”如同巨大的高墙，将曾经自由交流的计算机科学界分割成了一座座孤岛。

正是在这种工业化垄断的至暗时刻，麻省理工学院（MIT）人工智能实验室里的一位研究员，以一种近乎宗教般的执拗，吹响了软件复兴的反叛号角。他就是理查德·斯托曼（Richard Stallman）。

## 从一台施乐打印机谈起：道德的越界与开源的萌芽

斯托曼的反叛，起始于一件在商业社会中看似微不足道的小事。

当时，MIT 实验室引进了一台新型的施乐激光打印机。这台打印机极易夹纸，严重影响了研究人员的效率。按照以往的黑客习惯，斯托曼决定修改打印机的驱动程序代码，以便在夹纸时能自动给用户发送提醒。然而，当他向卡内基梅隆大学的同行索要这部分源码时，遭到了拒绝——这套代码被施乐公司烙上了“商业机密”印戳，工程师们签署了严格的保密协议（NDA）。

在绝大多数工程师看来，这不过是商业世界的游戏规则；但在斯托曼眼中，这是对人性与道德的背叛。本质上，他认为软件技术应该像数学定理或物理学定律一样，具有天然的公共属性。阻断了程序员互相帮助、互相改进程序的可能性，等同于在知识世界里设置了非正义的栅栏。

这种底层的哲学认知，构成了后来所有自由软件运动的基石。斯托曼不仅是一位追求极致的工匠，更是一位思想上的苦行僧。为了不受雇主的知识产权限制，他毅然辞去了 MIT 的公职，但他依然睡在实验室的行军床上，以此开始了他漫长得如同西西弗斯推石上山般的抗争。

## 宏大的乌托邦：GNU 计划的建立与“大宪章”GPL 协议

1983 年，斯托曼在新闻组正式发表了犹如檄文般的《GNU 宣言》，宣布要打造一个完全自由、开放且与 UNIX 兼容的操作系统。他将其命名为 **GNU**（GNU's Not UNIX 的递归缩写）。

客观地评价，GNU 计划是计算机科学史上一场了不起的民间基础设施建设。斯托曼不仅亲自上阵写出了令人叹为观止的 Emacs 文本编辑器，他还创立了自由软件基金会（FSF），并召唤全球认同他理念的黑客加入。他们从最底层的 C 语言编译器（GCC）、调试器（GDB）、函数库（glibc）开始，一砖一瓦地重新搭建大厦，这硬生生地从商业巨头的手指缝里，为全人类一点点夺回了工具的自由。

然而，仅有工具是不够的。资本天然具有兼并和掠夺的倾向，大公司完全可以将这些自由软件的代码拿去稍加修改，再以闭源的形式打包售卖。为了防止自由软件被商业窃取，斯托曼展露出了他除了技术狂热之外，极其高超的法律素养。

他与律师起草了著名的 **GPL（通用公共许可证）** 协议。GPL 巧妙地利用了现有的版权法（Copyright）体系，但将其逻辑完全反转，被黑客们戏称作“著佐权（Copyleft）”。它的核心条款是：你可以自由使用、修改这些代码，但如果你将修改后的软件对外分发，你就必须毫无保留地开放你的源代码，并且同样基于 GPL 协议。

某种意义上说，GPL 就是开源软件界的“传染性大宪章”。它如同一把达摩克利斯之剑，震慑住了那些试图将开源成果据为己有的贪婪掠夺者。这是商业史上一次极具想象力的法律黑客行为，它在资本壁垒森严的丛林中，生生辟出了一叶可以繁衍无数生命的避难舱。

## 宏伟蓝图的阿喀琉斯之踵

在 80 年代的最后几年，GNU 计划取得了巨大的成功，成体系的基础组件已经基本完备。但是，这座华丽的系统中，始终缺失了一块至关重要的核心——操作系统内核（Kernel）。

GNU 团队所设计的 Hurd 内核，由于过分追求理论上的完美，采用了当时极其先锋且复杂的“微内核（Microkernel）”架构。这种架构将众多功能从内核转移到用户空间，导致组件间的通信开销惊人，调试难度呈现指数级上升。在这个关键的战略节点上，斯托曼和他的精英团队陷入了严重的学术偏执之中，Hurd 内核长期在开发泥潭里挣扎，迟迟无法拼上整个系统的最后一块拼图。

历史证明，试图由一组自顶向下设计的精英团队去攻克一个脱离务实需求的完美理论架构，往往是迟缓且脆弱的。

在这个青黄不接的真空期，无数认同自由软件的极客只能在昂贵的商业计算机前望洋兴叹。他们如同拥有一整套顶级豪华装修材料的流浪汉，却找不到哪怕一块可以打地基的平地。

所有的引线都已铺设完毕，万钧雷霆的能量已经在 GPL 的庇护下积蓄了七年。现在，这一切都在等待一粒火星，等待一个能够被务实主义驱使、不去追求虚无完美的工程师的出现。
