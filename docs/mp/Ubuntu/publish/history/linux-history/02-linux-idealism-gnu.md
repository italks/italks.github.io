# Linux 演进史（二）：他为了修改打印机代码被拒绝，一怒之下辞职改变了世界

> **摘要**：一台施乐打印机引发的风波，让理查德·斯托曼辞去MIT的工作，发起GNU计划和GPL协议。他用法律武器对抗商业封闭，为开源运动奠定基石。GNU虽成果丰硕，却因内核Hurd追求完美而陷入泥潭。

**一台经常夹纸的施乐打印机，一个被拒绝的代码请求，一位倔强的程序员。这些看似微不足道的元素，竟然引发了改变整个软件产业历史的蝴蝶效应。**

1980年，麻省理工学院人工智能实验室。理查德·斯托曼（Richard Stallman）盯着眼前这台新引进的施乐激光打印机，陷入了沉思。

这台打印机有个让人抓狂的毛病——它特别喜欢夹纸。每当夹纸发生，用户们就像无头苍蝇一样在实验室里乱转，浪费时间。按照黑客们的传统，斯托曼想修改打印机驱动程序，让它能自动提醒用户。这不是什么高深的技术，只是几行代码的事。

**但当他向卡内基梅隆大学索要源代码时，得到的回复却是冷冰冰的三个字："商业机密"。**

工程师们签署了严格的保密协议（NDA），代码被施乐公司锁进了保险箱。在大多数人看来，这只是商业世界的游戏规则。但在斯托曼眼中，**这是对程序员灵魂的宣战**。

他看到的是一个可怕的图景：曾经自由分享代码的黑客社区正在崩塌，取而代之的是一道道封闭的商业围墙。每一家公司都在筑墙，每一行代码都被贴上价格标签。

**"如果软件不能被自由分享，那它就不是真正的知识，只是商品。"**

这个理念，将引发一场持续四十年的革命。

## 一个人的战争：从施乐打印机到 GNU 宣言

斯托曼做出了一个惊人的决定：辞去 MIT 的稳定工作，睡在实验室的行军床上，开始了一场孤独的抗争。

这听起来像是疯子的行为。在那个年代，软件工程师是令人羡慕的高薪职业。放弃铁饭碗，去追求什么"软件自由"，简直是不务正业。

**但斯托曼看到的，是别人看不到的未来。**

1983年，他在新闻组发表了著名的《GNU 宣言》。他宣布要打造一个完全自由、开放的操作系统，并将其命名为 **GNU**（GNU's Not UNIX 的递归缩写——一个典型的程序员式幽默）。

这不是一个简单的技术项目，而是一场哲学运动。斯托曼认为：**软件应该像数学定理或物理定律一样，属于全人类的公共知识。**

## 最聪明的"法律黑客"：GPL 协议的诞生

斯托曼的伟大之处，不仅在于他能写出令人叹为观止的 Emacs 编辑器和 GCC 编译器，更在于他发明了一套让资本家们都束手无策的法律武器——**GPL（通用公共许可证）**。

这是一个天才般的发明。GPL 巧妙地利用了现有的版权法（Copyright）体系，但将其逻辑完全反转，被黑客们戏称为"著佐权（Copyleft）"。

**它的核心条款简单而霸气：**
> 你可以自由使用、修改这些代码，但如果你将修改后的软件对外分发，就必须开放源代码，并且同样基于 GPL 协议。

**这就像是软件界的"传染病毒"——一旦使用了 GPL 代码，你的软件就必须开源。**

这个协议震慑住了那些试图将开源成果据为己有的大公司。它如同一把达摩克利斯之剑，在资本壁垒森严的丛林中，生生劈出了一片自由的净土。

## 完美的陷阱：GNU 的致命缺陷

到 1980 年代末，GNU 计划取得了惊人的成就。斯托曼和他的团队从零开始，构建了一整套强大的开发工具：
- **GCC**：世界上最强大的 C 语言编译器
- **Emacs**：功能强大到可以"做任何事"的编辑器
- **GDB**：调试器的标准
- **glibc**：核心函数库

**但有一个致命的问题：他们缺少最重要的东西——操作系统内核。**

GNU 团队设计的 Hurd 内核，因为过分追求理论完美，采用了当时最先进的"微内核"架构。这在学术上很美，但在工程上却是灾难。组件间复杂的通信让调试变得噩梦般困难，项目陷入了无休止的泥潭。

**斯托曼犯了一个精英主义者常犯的错误：试图用完美主义的蓝图，去建造一个复杂的现实系统。**

历史一次又一次证明：**最好的产品，往往不是最完美的设计，而是最早能用的方案。**

在这个青黄不接的时期，无数认同自由软件理念的程序员只能在昂贵的商业 UNIX 机器前望洋兴叹。他们就像拥有一整套顶级装修材料的流浪汉，却找不到一块可以打地基的土地。

**所有的基础设施都已就绪，只差最后一颗火种。**

而这个火种，即将在遥远的北欧点燃。
