# Ubuntu 演进史（七）：隐形的数字基石——主宰公有云、AI 算力与 Snap 生态之争

在过去的科技编年史中，我们往往倾向于用消费级终端设备（如个人电脑的屏幕或智能手机的外壳）的成败，去衡量一家科技公司的伟大程度。如果以此为标准，马克·沙特尔沃思创立 Canonical 并试图推翻微软在数亿办公桌上统治的“乌班图”桌面运动，确实是一场悲壮且未能如愿的堂吉诃德式冲锋。

但历史的迷人之处恰恰在于它不可预测的暗流涌动。正当外界惋惜 Ubuntu 在桌面和手机端偃旗息鼓时，如果你将视线从那些光鲜亮丽的消费电子展台移开，投向全球各大互联网科技巨头那轰鸣、冰冷且不见天日的数据中心机房，甚至是那些用于演算自动驾驶模型、破解生命密码的超级计算机集群时。

你才会惊愕地发觉，那个原本梦想只是给人类送去免费桌面系统的“疣猪”，早已登基成为了这个云计算与人工智能时代真正的幕后神明。

## 霸权基座：公有云与 AI 算力时代的绝对王者

在探讨 Ubuntu 为何能够统治当代服务器与云计算设施之前，我们要理解那个极其朴素的商业痛点：企业系统到底需要什么？

答案是：在海量的规模效应下，对**无缝升级（LTS）**与极其稳定的**软件依赖链**的极致渴求。

自 2006 年 6.06 Dapper Drake 版本起，Canonical 首创了后来改变整个服务器生命周期的 **LTS（Long Term Support，长期支持版）** 的节奏。每逢偶数年的 4 月份（例如 14.04、16.04、18.04...24.04），Ubuntu 便会推出一个提供长达 5 年甚者 10 年级别安全更新、且绝对不再轻易改动底层重大架构版本的基座核心。

这是一个具有降维打击意义的工程创新。在此之前，如果一家企业使用了那些迭代周期极短、喜欢随意更新内核并造成原本跑得好好的生产环境轰然崩溃的所谓开源发行版系统，运维工程师无异于在火山口上行走舞蹈。而那些高昂的商用系统，其授权费用又会随着云计算节点的爆炸式增长（成千上万台虚拟机实例起步）而让企业财报血本无归。

在 21 世纪的第二个十年，由于 Ubuntu 在底层系统模块上与大名鼎鼎的不垮神话 Debian 一脉相承，并且叠加了令公有云巨头如痴如醉的 LTS 免费战略承诺；当 AWS（亚马逊）、Azure（微软云）、GCP（谷歌云）这些云计算帝国的巨轮起航时，Ubuntu 顺理成章地成为了所有云计算租户（如早期的各种独角兽初创公司、硅谷巨头大本营）装载量占比第一大的首选镜像系统。在今天哪怕最具争议的云市场份额统计中，它毫无悬念地占据着整个全球公有云、云原生基础设施的半壁江山甚至更多。

随后爆发的 Docker（容器沙盒机制）、Kubernetes（多集群编排组件），甚至由于英伟达（NVIDIA）的 GPU 显卡在底层深度学习（CUDA 算子）的高度捆绑特性；由于在 Ubuntu 上拥有全天下最新且最被广泛验证的技术安装文档以及软件源包依赖构建环境。它成为了今天所有的深度学习平台算法训练机、智能汽车车机边缘计算乃至 AI（哪怕是开启大模型时代浪潮的 OpenAI 那极其恐怖的 H100 GPU 算力集群堆群后台操作），唯一的最优解底座。

它没有占领普通人的客厅和书房桌面，但它极其魔幻地承载了那些构建当今数字文明世界所需的全部后端算力重工业。

## 帝国的阴影：Snap 沙盒争议与社区自由的再博弈

然而，正如所有伟大的软件生态帝国在膨胀至一定体量后，必然面临着向中央控管靠拢的历史周期律一样；马克与他的 Canonical 在获得了如此空前的护城河后，也面临着如何在保证巨型企业客户极其敏捷分发的需求下获取合法商业回报的历史考题。

这就是近几年再次在整个极客圈掀起惊涛骇浪的 **Snap 软件包生态**争议。

从历史的技术进步角度来看，由于经典的 APT（Debian 体系的包管理）依然存在着依赖库牵一发动全身的“地狱（Dependency Hell）”隐患。Canonical 强制推广了一种名为 Snap 的沙盒容器型应用分发打包格式。只要软件开发者基于这套标准打包（哪怕内置了一整套臃肿的自身环境私有依赖），应用即可以在任何基于 Linux 底层的发行版本上不受干扰地一键部署甚至静默后台全自动更新。这本是在应用分发上为了抹平碎片化的绝佳利器。

但它的“致命原罪”在于其控制权的彻底封闭化垄断。有别于经典的开源去中心化架构仓库（即便有后来崛起的被社区狂热支持的独立方案 Flatpak 以及其公开可自建的机制）；Snap 的云端分发服务器核心（Snap Store）后端，是完完全全掌握在 Canonical 这家商业公司的闭源代码以及私人商业集群里的。不仅如此，Ubuntu 极其傲慢地在最近的版本发行中强行将系统的诸多基石软件（比如大家都十分依赖的火狐 Firefox 甚至后台诸多底层构件）剥离出原生开源社区架构，硬分发成了臃肿迟钝却无法被卸载替换的 Snap 沙盒。

这在一个充满自由反抗精神的所谓去中心化的“集市”里，等同于变相在开源世界的头顶建起了一座属于 Canonical 自己可以掌控生杀大权（且强行吃拿卡要所有后端分发数据流向）的“苹果式 App Store 壁垒”。

对于当年的理想主义者马克而言。无论它在企业级服务器集群管理上多么地完美且被商业财阀接纳，这也确实引发了 Linux 原教旨布道者群体的极其强烈的指责与愤怒。它就像是一句在历史长廊里回荡着极具讽刺意味的宿命诅咒：那个曾经屠龙的少年，在建立起庞大的数字基座帝国后，为了保证系统的极度统一连贯甚至未来的企业财报上市估值，最终也不可避免地长出了一层冰冷的、带着商业垄断气息的鳞片。

## 结语：科技史观下的星辰大海

如果以大历史的视野回顾这跌宕起伏的二十载《Ubuntu 演进史》。

它发源于南非这片与现代科技极度违和的贫瘠大陆上的一种“对他人仁慈”的图腾信仰；它脱胎于一位早早实现跨阶层财务自由并奔赴外太空俯瞰人类渺小后的非洲亿万富翁的那一场狂妄大梦。它曾在桌面千禧年的荒漠中以极其惊艳的光盘派送与 3D 火焰席卷全球，又曾在自傲的强推自研大一统风车之战中付出千万成本后撞得头破血流。

在这些充满了商业荒诞和技术极客妥协的大悲大喜背后。今天，当任何一个远在非洲甚至亚洲普通村镇且身无分文的小镇做题家，用一台哪怕是十几年前淘汰的最为破旧的电子垃圾二手笔记本，也能极其随意地插上一个 U 盘，几分钟后便能在屏幕上获取一个拥有人类算力巅峰、能编写并撬动最前沿的神经网络甚至宇宙暗物质模拟的完整、免费且全能的系统级开发底座时。

那个二十年前的马克，与无数为了这句“为人类打造的系统（Linux for Human Beings）”而耗干了青春的无名黑客们。虽然他们丢失了攻下微软和苹果城池的终极权杖，但他们不仅早已如数地兑现了当年的那句誓言，而且也真地犹如那个承载着非洲传统图腾的单词一般，硬生生地用他们的技术与妥协所化作的砖石，成为了垫起这个人类信息数字化狂潮中最沉稳不拔的那一块不可见的基石。

那是一篇极其浩瀚悲壮、且必将被永远铭记的开源星辰大海物语。
