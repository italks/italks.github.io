# Ubuntu 演进史（二）：为人类打造的 Linux——Canonical 创立与第一只“疣猪”的诞生

从操作系统百年的演进周期来看，底层代码的健壮仅仅是技术世界里的一张入场券。真正能够改变普通人生活方式的，往往是将这层坚硬的技术外壳包裹进一层顺滑乃至充满人情味的交互体验之中。

2004 年的 Linux 桌面世界，正处于一种极度撕裂的状态。一方面，在服务器端它已经锋芒毕露，但在个人电脑领域，它依然是只属于极客的玩具。如果你想在电脑上安装网卡驱动，或是想听一首歌，往往需要在一片全黑的终端里敲打数百行晦涩的命令。当时开源界最纯正、最受尊重的发行版 Debian，虽然坚如磐石，但对新手的态度却极其傲慢。

刚从太空返回地球的亿万富翁马克·沙特尔沃思发现了一个极其荒谬的悖论：自由软件运动的初衷是让“全人类”共享技术的自由，但它反人性的高操作门槛，却在客观上把 99% 的普通人类阻挡在了自由的大门之外。如果不打破这种极客的精英主义壁垒，操作系统的版图将永远被微软的 Windows 甚至苹果统治。

## 伦敦公寓里的密谋与 Canonical 的诞生

在商业世界里，用一套金字塔式的“大教堂”工业体系去重构一套“集市”产品的用户体验，是需要极大勇气的。2004 年 4 月，马克在伦敦市中心租下了一间公寓。他利用自己在开源社区的声望与雄厚的财力，将当时 Debian 社区里最顶尖的几位黑客空运到了这间公寓里召开闭门会议。

这群人给自己起了一个极其随意的项目代号：`no-name-yet.com`。

在这场被后人无数次提起的公寓聚会中，马克正式出资成立了一家名为 **Canonical** 的公司（意为“经典的、规范的”）。这绝不是一家普通的唯利是图的科技企业，而更像是一支带着强烈理想主义色彩的数字远征军。马克为这支部队制定了三条极其狂妄但也极其致命的作战纲领：

首先，**极致的开箱即用（Out of the box）**。系统必须能自动识别尽可能多的声卡、显卡和无线网卡。安装完系统的第一秒，用户就应该能打开浏览器上网、用文档编辑器打字，而不需要查阅任何超过一页纸的说明书。“为人类打造的 Linux（Linux for Human Beings）”不仅是一句口号，而是一套严苛的产品哲学。

其次，**承诺永远免费**。不仅系统本体免费，任何后续的升级版本也将彻底免费。这在当时动辄需要数百美金购买系统光盘以及付费升级驱动的年代，无异于对传统软件商业模式的一次维度打击。

第三，也是最令当时的开源界震惊的一点：**严格遵守每六个月一次的发布周期**。在当时的开源圈，由于开发者都是凭兴趣兼职，“什么时候修完 Bug 什么时候发布（When it's ready）”是金科玉律。Debian 有时甚至会因为追求完美而连续跳票两三年。但马克坚持像制造工业品一样约束发行日期——每年的 4 月和 10 月必须发布新版。这种犹如钟表般精准的节奏，瞬间在企业和个人用户心中建立起了无可替代的确定感。

## 第一只改变世界的“疣猪”

半年之后的 2004 年 10 月，这套被正式定名为 Ubuntu 的操作系统发布了它的第一个版本：**Ubuntu 4.10**，代号“疣猪（Warty Warthog）”。

在软件史上，这个版本是极具象征意义的。从底层上看，它直接抽取了 Debian 非常稳定的软件源作为底座，这不仅避免了重复造轮子，更继承了 Debian 强大的依赖包管理系统（APT）。但在用户感知层，Canonical 的工程师们花费了巨大的心血去打磨 GNOME 桌面、编写易于理解的图形化安装向导，并将大量繁琐的配置文件封装在后台自动执行。

那是一次让无数老极客大跌眼镜的发布。当一张薄薄的安装光盘被塞入随意一台兼容机，伴随着几下鼠标点击，不到二十分钟，一个带着极具非洲风情的棕色背景桌面便跃然屏幕之上。没有恼人的命令行报错，声卡正常发声，网卡顺利连网。

这第一声粗野但充满生命力的“猪叫”，彻底劈开了 Linux 与普通用户之间那扇上锁的大门。这不仅是马克的一次工程实验的成功，更是开源文化在普及之路上的一次伟大觉醒。然而，仅仅做到易用是不够的，要想在全球范围内彻底点燃这场“乌班图”烈火，马克还需要跨越当时发展中国家的一道物理鸿沟——宽带网络。
