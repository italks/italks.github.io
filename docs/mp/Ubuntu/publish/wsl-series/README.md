## 📖 系列简介

### 什么是 WSL？

**Windows Subsystem for Linux（WSL）** 是微软在 Windows 10/11 上提供的兼容层，让你无需安装虚拟机就能原生运行 Linux 环境。自 2016 年首次发布以来，WSL 已经成为数百万开发者在 Windows 上体验 Linux 的首选方案。

### 为什么学习 WSL？

| 用户类型 | WSL 解决的痛点 |
|:---|:---|
| **Windows 开发者** | 无需双系统即可使用 Linux 工具链 |
| **Linux 爱好者** | 在保留 Windows 的同时享受 Linux 命令行 |
| **后端工程师** | 一套机器同时拥有两个操作系统生态 |
| **DevOps/运维** | 在本地测试 Shell 脚本和部署流程 |
| **学生/初学者** | 零成本入门 Linux，不影响日常使用 |

### WSL 版本对比

| 特性 | WSL 1 | WSL 2 |
|:---|:---:|:---:|
| 架构 | 系统调用翻译 | 完整 Linux 内核（虚拟化） |
| 兼容性 | 约 95% | 接近 100% |
| 文件系统性能 | **更快**（原生 NTFS 访问） | 较慢（通过 9P 协议） |
| Docker 支持 | ❌ 不支持 | ✅ 完整支持 |
| 内存占用 | 较低 | 较高（需分配虚拟内存） |
| 启动速度 | 快 | 稍慢（需启动 VM） |
| 适用场景 | 轻量级工具、脚本开发 | 完整 Linux 环境、容器开发 |

## ✅ 学习前提

- **硬件要求**：
  - Windows 10（版本 2004+，内部版本 19041+）或 Windows 11
  - 至少 4GB RAM（推荐 8GB+）
  - 启用虚拟化（BIOS 中开启 VT-x / AMD-V）

- **知识储备**：
  - 基本的 Windows 操作能力
  - 对 Linux 有初步了解（或愿意边学边用）
