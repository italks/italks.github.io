# 02. 准备工作与安装方式选择

> **摘要**：在安装 Ubuntu 前，本文帮你做好硬件检查与方案选择。涵盖 CPU、内存与硬盘的配置要求，对比虚拟机、双系统与单系统的优劣，助你根据需求（体验或主力）选择最稳妥的安装路径，将折腾风险降至最低。

![封面图](../../imgs/ubuntu-desktop-guide-02-cover.svg)

这一章不讲“怎么装”，只做三件事：选方案、做准备、把风险压到最低。

- 你想先体验还是直接上主力
- 你能不能接受重启切系统（双系统）或直接替换（单系统）
- 你愿不愿意为了省事，先走虚拟机/WSL2 这条“安全路径”

## 1. 硬件要求检查

Ubuntu 24.04 LTS 虽然比 Windows 轻量，但仍有一定的硬件要求以保证流畅体验：

*   **处理器 (CPU)**：2 GHz 双核或更好；越新越省心（尤其是 Wi‑Fi/蓝牙驱动）。
*   **内存 (RAM)**：4 GB 能用；8 GB 起更像“主力桌面”；16 GB 适合多任务/开发/容器。
*   **硬盘空间**：25 GB 是“能装上”；50 GB 起才够日常；打算装 Docker/开发依赖建议更多。
*   **其他**：USB 接口（做启动盘）+ 稳定网络（安装时拉更新、装驱动）。

如果你打算走 **虚拟机**：重点看两点——内存够不够、CPU 虚拟化有没有开（Intel VT-x / AMD-V）。

## 2. 安装方式大比拼：哪种适合你？

先看结论：新手最稳的是 **虚拟机**；开发者在 Windows 上用命令行工具，最省事的是 **WSL 2**；要把 Ubuntu 当主力桌面，最终还是 **双系统/单系统**。

| 方式 | 适合谁 | 优点 | 缺点 | 推荐程度 |
| --- | --- | --- | --- | --- |
| 虚拟机 | 想体验、怕分区、做测试 | 不动硬盘分区；坏了删掉重来 | 性能打折；显卡/外设能力有限 | 新手首选 |
| WSL 2 | Windows 主力 + 需要 Linux 命令行 | 启动快；与 Windows 文件互通 | 更像“Linux 环境”而不是“完整桌面系统” | 开发者首选 |
| 双系统 | 既要 Linux 性能，又要 Windows 兼容 | 性能满血；两边互不影响 | 分区/引导有风险；切换要重启 | 主力桌面推荐 |
| 单系统 | 已不依赖 Windows 生态 | 最纯粹、最省资源 | 回头成本高；少数软件需要替代方案 | 明确需求再上 |

### A. 虚拟机安装 (Virtual Machine)
**适合人群**：完全的新手、想体验 Linux 但不想改变现有系统、开发测试。
*   **原理**：在 Windows/macOS 上安装一个软件（如 VMware, VirtualBox），在软件里运行 Ubuntu。
*   **优点**：最安全，弄坏了删掉重来即可，不影响主系统；可以同时运行 Windows 和 Ubuntu。
*   **缺点**：性能有损耗，体验不如真机流畅。

### B. WSL 2 (Windows Subsystem for Linux)
**适合人群**：开发者、需要在 Windows 下使用 Linux 命令行工具的人。
*   **原理**：微软官方提供的在 Windows 内核上运行 Linux 的子系统。
*   **优点**：与 Windows 深度集成，文件互通，启动速度极快，性能接近原生。
*   **缺点**：虽然支持 GUI 应用，但主要优势还是在命令行和开发环境，桌面体验不如完整版 Ubuntu。

### C. 双系统 (Dual Boot)
**适合人群**：游戏玩家（Windows 玩游戏）、既要 Linux 性能又要 Windows 兼容性的用户。
*   **原理**：将硬盘划分为两个区域，分别安装 Windows 和 Ubuntu。开机时选择进入哪个系统。
*   **优点**：Ubuntu 独占硬件资源，性能满血；互不干扰。
*   **缺点**：安装步骤稍繁琐，分区操作有风险（需备份数据），切换系统需要重启。

### D. 单系统 (Single Boot)
**适合人群**：Linux 铁粉、老旧电脑复活、开发者。
*   **原理**：格式化整块硬盘，只安装 Ubuntu。
*   **优点**：最纯粹的 Linux 体验，磁盘空间最大化。
*   **缺点**：彻底告别 Windows（除非重装），部分 Windows 独占软件可能无法运行（需用 Wine）。

**新手建议**：先试 **虚拟机** 或 **WSL 2**，把软件安装、输入法、基本命令用顺；确认要长期用，再上 **双系统**。

## 3. 开装前必做清单（双系统/单系统强烈建议）

这一段看起来啰嗦，但它能帮你避开 80% 的“装完开不了机/进不去 Windows”。

**数据与系统层面**

1. 备份重要数据：至少把桌面/文档/照片/浏览器书签导出一份到移动硬盘或云盘。
2. 记下 Windows 的 BitLocker/设备加密恢复密钥（如果启用过）。
3. 关闭 Windows 快速启动（否则容易出现分区被占用、无法挂载的问题）。
4. 给双系统预留空间：在 Windows 磁盘管理里先压缩出一块“未分配空间”（更可控）。

**固件与启动层面**

1. 确认机器用的是 UEFI 模式（近几年基本都是）。
2. Secure Boot（安全启动）一般不需要关；但如果你用的是 NVIDIA 独显且驱动折腾较多，遇到问题再考虑调整。
3. 准备好启动菜单快捷键（F12/F9/Esc/Del 等，下一章会更细讲）。

## 4. 准备安装介质（双系统/单系统）

如果你选择了双系统或单系统安装，你需要制作一个 USB 启动盘。

### 步骤一：下载 Ubuntu 镜像
访问 [Ubuntu 官网](https://ubuntu.com/download/desktop) 下载最新的 LTS (长期支持) 版本 ISO 文件。

如果你网络条件一般，建议顺手找一个就近的镜像站点下载，同一个版本的 ISO 是一致的。

### 步骤二：准备一个 U 盘
准备一个至少 8GB 的 U 盘。**注意：制作过程会清空 U 盘数据，请提前备份！**

### 步骤三：使用工具写入镜像
推荐使用以下工具将 ISO 写入 U 盘：
*   **Rufus (Windows 推荐)**：小巧、免费、开源。
    1.  插入 U 盘，打开 Rufus。
    2.  "设备" 选择你的 U 盘。
    3.  "引导类型选择" 点击 "选择"，找到下载的 Ubuntu ISO 文件。
    4.  分区类型一般选 GPT（对应 UEFI），目标系统选 UEFI（如果能选到）。
    5.  其他保持默认，点击 "开始"。
    5.  选择 "以 ISO 镜像模式写入"。
*   **BalenaEtcher (跨平台)**：界面美观，操作极其简单（Select Image -> Select Drive -> Flash）。

等待写入完成，你的 Ubuntu 安装盘就制作好了！
