# Linux 7.0 即将到来：5个让 Ubuntu 26.04 值得期待的内核新特性

Linus Torvalds 最近在发布 Linux 7.0-rc4 时说了一句很有意思的话：

> "rc4 比正常阶段大很多，但我查了查，实际的代码变更看起来小而分散，大多是良性修改。也许开发者在主版本号升级时格外活跃吧。"

从 6.x 跳到 7.0，不只是数字变了。

Linux 7.0 的合并窗口引入了一些**真正有里程碑意义的变更**。而 Ubuntu 26.04 LTS 将搭载这个新内核（或很快通过 HWE 内核流更新），这意味着这些变化会直接影响数百万 Ubuntu 用户。

今天我们挑出最重要的 5 个新特性，用大白话帮你拆解。

---

## 一、Rust 正式"毕业"：不再是实验品

这是 Linux 7.0 最具标志性的事件。

### 发生了什么？

Linux 内核中一直存在一个标记：Rust 支持"**实验性**"（EXPERIMENTAL）。这个标签从 Rust 第一次进入内核时就存在，提醒所有人——这还在试验阶段。

**Linux 7.0 正式移除了这个标签。**

Rust for Linux 的核心贡献者 Miguel Ojeda 在提交信息中写道：

> "实验已经完成。也就是说，Rust 会留在这里。我希望这能向公司和其他组织表明内核的承诺。"

### 这意味着什么？

| 之前 | 现在 |
|:---|:---|
| Rust 在内核中是"实验性"功能 | Rust 是内核的**正式组成部分** |
| 企业犹豫是否投入资源 | 官方承诺长期支持，企业可以放心投入 |
| 各子系统维护者可以拒绝 Rust 补丁 | 仍可自行决定，但官方态度已经明确 |

### 实际影响

别以为这只是"文档改了几行字"。这个变化有实际的连锁反应：

- **数百万台 Android 设备**已经在用包含 Rust 代码的内核
- 部分主流 Linux 发行版已经默认包含 Rust 内核组件
- 内核为 **Rust 1.95** 做了兼容准备，工具链更加成熟
- 引入 `__rust_helper` 标注，改善了内核 LTO（链接时优化）时的构建体验

### 对开发者的意义

如果你是内核开发者或者对底层编程感兴趣，现在是学 Rust 的好时机。内核已经明确表示：**Rust 不是过客，而是长期居民。**

## 二、Btrfs 和 XFS：文件系统变强了

### Btrfs：自愈能力上线

Btrfs 在 Linux 7.0 中合并了**自主自愈支持**。

简单说：当 Btrfs 检测到数据损坏时，可以**自动从冗余副本中修复**，不需要人工干预。

对于使用 Btrfs 作为根文件系统的用户（比如 Fedora 默认就用 Btrfs），这相当于多了一层自动保险。

### Btrfs 直接 I/O 增强

- 支持在**块大小超过页面大小时**使用直接 I/O
- 文件系统现在可以在直接 I/O 期间使用**回弹缓冲区**来计算校验和
- 对数据库等大块 I/O 场景有性能提升

### XFS：同样获得自愈能力

XFS 作为企业级文件系统，也合并了自主自愈支持。如果你在服务器上用 XFS，数据安全性又上了一个台阶。

### EROFS：压缩优化

EROFS（主要用于 Android 和嵌入式设备的只读文件系统）也获得了改进：

- **默认启用 LZMA 压缩**
- DEFLATE 和 Zstandard 压缩不再标记为实验性
- 支持跨 EROFS 文件系统**共享页面缓存**，减少内存占用

### 对 Ubuntu 26.04 的影响

Ubuntu 26.04 默认使用 ext4，但这些文件系统的改进对特定场景很重要：

- 如果你用 Btrfs 做 NAS 或备份分区，自愈功能很实用
- 服务器用户在 XFS 上运行数据库会更安心
- EROFS 的优化对容器镜像存储有间接帮助

## 三、安全硬核升级：后量子密码学进入内核

这是 Linux 7.0 中**最前沿**的安全特性。

### 后量子签名：ML-DSA

Linux 7.0 首次在内核中支持 **ML-DSA 后量子签名算法**，并可用于**内核模块认证**。

为什么这很重要？

现在量子计算发展很快。虽然通用的量子计算机还远未成熟，但"先收集密文、以后用量子计算机破解"的攻击方式（称为"先存储后解密"）已经被安全界认真对待。

ML-DSA 是美国 NIST 标准化的后量子密码算法之一，被认为是**抗量子攻击**的安全选择。

Linux 7.0 把它引入内核，意味着内核模块签名可以在未来抵御量子攻击。

### 其他安全改进

| 特性 | 说明 |
|:---|:---|
| **移除 SHA-1 内核模块签名** | SHA-1 已不安全，但已签名的旧模块仍可加载（向后兼容） |
| **SELinux 控制 BPF 令牌** | 加强对 eBPF 程序的安全管控 |
| **NETFILTER 审计增强** | 网络防火墙审计记录现在包含源和目标端口号 |

### SHA-1 淘汰的意义

SHA-1 在密码学领域已经被认为不安全。Linux 7.0 虽然不再允许**新签名**使用 SHA-1，但**已签名的旧模块**仍然可以加载——这是一个平滑过渡的设计。

如果你自己编译内核模块，需要改用更强的签名算法（如 SHA-256 或 ML-DSA）。

## 四、AccECN 默认启用：网络性能优化

### 什么是 AccECN？

AccECN（Accurate Explicit Congestion Notification，精确显式拥塞通知）是一种改进的 TCP 拥塞控制机制。

传统 ECN 只能告诉发送端"网络拥堵了"（1 bit 信息）。而 AccECN 能提供**更精确的拥堵程度信息**。

### Linux 7.0 的变化

Linux 7.0 成为**首个默认启用 AccECN 的操作系统版本**。

### 谁会受益？

主要受益场景是**数据中心和多流量竞争环境**：

- 多个应用同时抢占带宽时，AccECN 能更精确地分配带宽
- 与现代智能网卡更好协同
- 高密度容器部署环境中，网络延迟更稳定

### 对普通用户的影响

如果你只是日常上网、看视频，感知可能不大。但如果你：

- 运行 Kubernetes 集群
- 在服务器上部署微服务
- 使用高密度 Docker 环境

AccECN 默认启用会带来可感知的网络性能提升。

## 五、调度器和性能：更聪明的 CPU 管理

### 调度器简化

Linux 7.0 的进程调度器简化了抢占模式：

- 只保留 **PREEMPT_LAZY** 和 **PREEMPT_FULL** 两种模式
- 淘汰了部分旧的抢占模式
- 代码更简洁，维护更容易

### io_uring 增强

io_uring 是 Linux 近年来最成功的高性能 I/O 接口。7.0 中的改进：

- 支持 **BPF 过滤 io_uring 操作**，增强沙箱能力
- 引入**非环形队列选项**，提升缓存性能

如果你用高性能存储（NVMe SSD）或高并发应用，io_uring 的这些改进会带来实际收益。

### BPF 类型格式（BTF）加速

BPF 程序加载时使用的 BTF 查找，现在改用**二分查找**，加载速度更快。

对大量使用 eBPF 的场景（如 Cilium 网络策略、Falco 安全监控），这是一个实实在在的性能优化。

### 容器启动加速

新增 `OPEN_TREE_NAMESPACE` 选项，可以**快速创建新挂载命名空间**（不需要克隆现有命名空间）。

这对容器启动速度有直接提升——创建容器时最耗时的操作之一就是设置挂载命名空间。

### 每CPU缓存优化

Sheaf 缓存改为**每 CPU 缓存**（per-CPU cache），减少了跨 CPU 的缓存争用。

在多核服务器上，这个优化能降低延迟，提高吞吐量。

---

## 其他值得关注的变化

### Arm 64位原子操作优化

支持在提供该功能的 Arm CPU 上进行原子 64 字节加载和存储，对 ARM 服务器有性能提升。

### initrd 简化

移除了基于旧的 `linuxrc` 接口的 initrd 代码，计划在 2027 年完全转向 initramfs。这会简化启动流程。

### nullfs 文件系统

新增空文件系统 `nullfs`，可用作根文件系统，简化启动过程。这对容器和虚拟化场景有用。

### RCU 任务跟踪重新实现

RCU（Read-Copy-Update）是内核中最重要的并发原语之一。7.0 重新实现了 RCU 任务跟踪机制，弃用了旧的 trace 接口，代码更清晰。

## 各发行版搭载时间预估

| 发行版 | 预计搭载时间 | 说明 |
|:---|:---|:---|
| Arch Linux | 2026年4月底-5月 | 滚动更新，最快跟进 |
| Fedora 44 | 2026年5月 | 目标搭载最新稳定内核 |
| Ubuntu 26.04 LTS | 2026年5-6月 | 通过 HWE 内核流更新 |
| RHEL 10 / AlmaLinux | 2027年或更晚 | 企业版采用向后移植模型 |

Ubuntu 26.04 LTS 发布时可能搭载的是 Linux 6.20（最终稳定版），Linux 7.0 会通过 **HWE（Hardware Enablement）内核流**在发布后几个月内推送。

所以别急着等 Linux 7.0——安装 Ubuntu 26.04 后，通过简单的命令就能升级到新内核：

```bash
# 安装 HWE 内核（发布后可用）
sudo apt install linux-generic-hwe-26.04

# 或者通过 mainline PPA 提前体验
sudo add-apt-repository ppa:cappelikan/ppa
sudo apt install mainline
```

## 总结

Linux 7.0 的 5 个核心变化：

1. **Rust 正式化**：从实验品变成内核长期居民
2. **文件系统增强**：Btrfs/XFS 自愈、Btrfs 直接 I/O 优化
3. **后量子密码学**：ML-DSA 签名进入内核，面向未来安全
4. **AccECN 默认启用**：数据中心网络性能提升
5. **调度器和性能优化**：io_uring、BPF、容器启动全面提速

**Linux 7.0 + Ubuntu 26.04 LTS = 2026年最值得期待的桌面/服务器组合。**

内核虽然"看不见"，但它是一切的基础。这些改进会在你日常使用中悄悄发挥作用——更快的文件操作、更稳的游戏帧率、更安全的系统。
