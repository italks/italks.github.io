Ubuntu 26.04 安装前必做：5个关键准备,避开90%的坑

你可能听过这样的故事：

有人装完 Ubuntu 发现网卡驱动没有,只能用手机热点下载驱动;
有人分区时手一抖,把 Windows 系统盘格式化了;
还有人装完才发现常用软件找不到替代品,又灰溜溜切回 Windows。

安装 Linux 最怕的不是技术难度,而是那些"没想到"的坑。

Ubuntu 26.04 将在4月正式发布,不少朋友已经跃跃欲试。但作为过来人,我得提醒你:
安装前的准备工作,往往比安装本身更重要。

这篇文章整理了 5 个关键准备步骤,都是真实踩坑经验的总结。
把这些做好,你的安装过程才能真正"丝滑"。

准备一:硬件兼容性自检

这是最容易忽略,也最容易翻车的环节。

为什么要检查?

很多人以为 Linux 硬件兼容性已经很好了,确实如此——但"很好"不等于"完美"。

最容易出问题的是这些:

  • 显卡驱动: NVIDIA 显卡需要闭源驱动,部分新显卡可能支持不完善
  • 无线网卡: 尤其是 Realtek 芯片,有些型号需要手动编译驱动
  • 蓝牙模块: 某些笔记本的蓝牙固件需要额外处理
  • 触控板: 部分高端笔记本的手势识别可能不完整

如何检查?

方法一:查看官方硬件兼容性列表(HCL)

Ubuntu 官方维护了一份认证硬件列表,可以在认证页面搜索你的设备型号。

方法二:Live USB 试用模式

这是最可靠的方法。用 U 盘启动后选择"试用 Ubuntu",不要急着安装。
在这个模式下测试:

  • 能否连接 WiFi
  • 声音是否正常
  • 外接显示器能否识别
  • 睡眠/唤醒是否稳定

方法三:搜索自己的硬件型号

在搜索引擎或论坛搜索:你的硬件型号 + Ubuntu 26.04。
如果有人遇到问题,通常能找到解决方案或确认是否支持。

小建议

如果你用的是比较新的硬件(比如刚发布的显卡或笔记本型号),建议等 Ubuntu 26.04.1 LTS 发布后再升级。
这个版本通常会修复首批用户反馈的硬件兼容问题。

准备二:数据备份方案

这个建议听起来很基础,但真的有太多人栽在这里。

必备备份清单

备份内容 备份方式 推荐工具
个人文档 云同步/外置硬盘 Nextcloud、rclone
浏览器数据 账号同步 Firefox Sync、Chrome Sync
开发环境配置 配置文件导出 dotfiles(.bashrc、.vimrc 等)
虚拟机/容器 导出镜像 Docker save、VirtualBox 导出
SSH 密钥 复制到安全位置 ~/.ssh 目录

开发者特别注意

如果你是开发者,这些文件一定要备份:

# 打包常用配置文件
tar -czf ~/config-backup.tar.gz \
  ~/.bashrc \
  ~/.zshrc \
  ~/.vimrc \
  ~/.ssh \
  ~/.gitconfig \
  ~/.config/Code  # VS Code 配置

Docker 用户记得导出重要镜像:

# 查看本地镜像
docker images

# 导出需要的镜像
docker save -o myimage.tar 镜像名:标签

备份验证

备份完成后,别急着删除原文件。
随机抽取几个重要文件,确认能正常打开和使用。

建议保留两份备份:一份云端,一份本地硬盘。
硬盘可能坏,云盘可能倒闭,双重保险最稳妥。

准备三:分区规划策略

分区是新手最容易纠结的环节。这里给你三个方案,按需选择。

方案一:简单方案(新手推荐)

适用场景:第一次装 Linux、硬盘空间较小(≤128GB)

/        整个硬盘(除 swap)
swap     与内存相当

优点:简单,不用纠结
缺点:重装系统会丢失所有数据

方案二:推荐方案(日常使用)

适用场景:日常办公和开发

EFI     512MB     (UEFI 启动必须)
/       50GB      (系统和软件)
/home   剩余空间  (个人数据)
swap    8GB       (休眠需要)

优点:重装系统不会丢失个人文件
缺点:需要提前规划空间

方案三:进阶方案(服务器/高级用户)

适用场景:服务器或对系统管理有要求的用户

EFI     512MB
/       30GB
/home   剩余空间的60%
/var    20GB      (日志和临时文件)
swap    与内存相当

优点:各分区独立,日志不会撑爆系统
缺点:规划复杂,空间利用率稍低

分区避坑指南

坑一:忘记 EFI 分区

UEFI 启动必须要有 EFI 分区,建议 512MB,格式为 FAT32。
如果忘记创建,系统可能无法启动。

坑二:swap 太大或太小

传统建议是 swap 设为内存的 1-2 倍,但在 SSD 时代,这个规则已经过时。

现在的建议:

  • 8GB 以上内存:swap 设 4-8GB 即可
  • 需要休眠功能:swap 设为与内存相当

坑三:/home 分区太小

很多软件的配置文件、缓存都在 ~/.config 和 ~/.cache 目录下。
如果 /home 分区太小,用久了会发现空间不够用。

建议至少预留 100GB 给 /home(如果你硬盘够大)。

准备四:启动盘制作要点

启动盘制作看似简单,但细节决定成败。

工具选择

Windows 用户:Rufus(推荐)

优点:界面简洁,功能完善,支持 UEFI
下载:https://rufus.ie

macOS/Linux 用户:balenaEtcher

优点:操作简单,界面美观
下载:https://www.balena.io/etcher/

多系统用户:Ventoy

优点:一个 U 盘装多个系统 ISO,无需重复格式化
官网:https://www.ventoy.net

关键设置

1. 选择正确的分区方案

现代电脑基本都是 UEFI 启动,所以选择 GPT 分区方案。

2. 关闭 Secure Boot

部分主板需要关闭 Secure Boot 才能启动 Linux。
具体方法各品牌主板不同,一般在 BIOS 的 Security 或 Boot 菜单中。

3. 验证 ISO 校验和

下载完成后,一定要验证镜像完整性:

# 验证 SHA256 校验和
sha256sum ubuntu-26.04-desktop-amd64.iso

对比官网提供的校验和,完全一致才算下载成功。
这个步骤能避免因镜像损坏导致的各种奇怪问题。

U 盘选择建议

  • 容量至少 8GB(推荐 16GB)
  • 读写速度不要太慢,否则启动会很慢
  • 使用 USB 3.0 接口的 U 盘体验更好

准备五:安装后必做清单预览

别以为装完系统就万事大吉了,还有一堆事等着你。

安装后第一步

更新系统:

sudo apt update
sudo apt upgrade -y

这步能修复已知问题,更新安全补丁。

安装常用软件:

每个人需要的软件不同,但有一些是通用的:

# 中文输入法
sudo apt install ibus-pinyin

# 压缩工具
sudo apt install p7zip-full unzip

# 多媒体编解码器
sudo apt install ubuntu-restricted-extras

# Git
sudo apt install git

# 基础开发工具
sudo apt install build-essential

提前准备

列出自己的软件清单:

把你在 Windows 上常用的软件列出来,逐一找到 Linux 替代品:

Windows 软件 Linux 替代品
Office LibreOffice、WPS
Photoshop GIMP、Krita
Notepad++ VS Code、Sublime Text
utools Albert、Ulauncher

记录网络配置:

如果你用的是静态 IP 或特殊网络配置,记得提前记录:

  • IP 地址
  • 网关
  • DNS 服务器
  • 代理设置

这些信息重装后需要重新配置。

总结

安装 Ubuntu 26.04 前的 5 个关键准备:

  1. 硬件兼容性自检:避免装完发现驱动不能用
  2. 数据备份:双重备份最稳妥
  3. 分区规划:/home 单独分区能救命
  4. 启动盘制作:验证校验和,选择正确分区方案
  5. 安装后清单:提前准备软件列表和网络配置

准备工作做得越充分,安装过程就越顺利。

与其在遇到问题时焦头烂额,不如花半小时把准备工作做扎实。
