# Linux 7.1 RC1 来了：NTFS写入暴涨110%，40年i486正式退役，你的电脑受影响吗？

> **阅读时长**：6分钟 | **难度**：⭐⭐⭐ 中级 | **关键词**：Linux内核、NTFS驱动、i486退役、硬件兼容性

---

Linux 内核开发从来不会让人无聊。

就在昨天（4月27日），Linus Torvalds 亲手发布了 **Linux 7.1-rc1**——7.1 系列的第一个候选版本。这意味着合并窗口关闭，新特性全部锁定，预计 **6月中旬** 正式版与我们见面。

但这次更新不一般。两个消息格外引人注目：

1. **NTFS 驱动被彻底重写，写入性能暴涨 35%~110%**
2. **运行了 40 年的 i486 架构，正式从 Linux 内核中移除**

一个是"新生"，一个 是"告别"。今天我们就来聊聊 Linux 7.1 到底带来了什么，以及**对 Ubuntu 用户意味着什么**。

---

## 一、NTFS 的复兴：从"能读"到"飞写"

### 为什么 NTFS 这么重要？

如果你用过 Linux，你一定有过这种经历——插上一个 NTFS 格式的移动硬盘或 U 盘（Windows 默认格式），Linux 能读，但写入速度感人，或者干脆各种报错。

这是因为 Linux 内核自带的 NTFS 驱动（NTFS3）一直是个"半成品"。它主要支持读取，写入功能孱弱且不稳定。这也是为什么很多人在双系统环境下不得不借助第三方工具（如 ntfs-3g）来管理 Windows 分区。

### 这次重写了什么？

在 Linux 7.1 中，来自韩国三星的内核开发者 **Namjae Jeon**（他也是 exFAT 驱动的作者）对 NTFS 驱动进行了**彻底重写**：

| 特性 | 旧驱动 (NTFS3) | 新驱动 (NTFS4) |
|:---|:---|:---|
| 写入支持 | 基础支持，性能差 | **完整写入，延迟分配** |
| 多线程写入 | 慢，锁竞争严重 | **提升 35%~110%** |
| 挂载 4TB 硬盘 | 慢 | **速度 ×4** |
| xfstest 测试通过 | 273 项 | **326 项 (+19%)** |
| 内核集成度 | 较低 | 深度整合 iomap + folio |

Linus 本人对这次更新给出了极高评价，称之为 **"NTFS Resurrection"（NTFS 复兴）**。

### 对 Ubuntu 用户的实际影响

👉 **WSL 用户狂喜**：WSL（Windows Subsystem for Linux）依赖 NTFS 文件系统来访问 Windows 文件。新驱动意味着 WSL 的文件操作性能将显著提升，尤其是代码编译、Docker 操作等磁盘密集型场景。

👉 **双系统用户受益**：如果你同时使用 Windows 和 Ubuntu，经常需要在两个系统间传文件，新的 NTFS 驱动将让你的跨系统文件管理体验大幅改善。

👉 **服务器/NAS 管理员注意**：对于需要挂载 NTFS 存储的服务器场景，新驱动的稳定性和性能都是重大利好。

> **如何启用？** 新旧驱动将在内核中共存一段时间，用户可通过 `NTFS_FS` 的 Kconfig 开关选择启用新驱动。

## 二、一个时代的终结：i486 正式退役

### 什么是 i486？

Intel 486 处理器于 **1989 年**问世，2007 年停产。它在 PC 历史上具有里程碑意义——第一款真正让家用电脑"能用起来"的处理器。

而 Linux 内核从诞生的第一天起，就一直支持 i486。直到今天。

但在 Linux 7.1 中，这个持续了 **40 年**的支持终于画上了句号。

### 具体移除了什么？

以下配置选项已被删除：
- **M486**：经典 486DX/DX2/DX4 系列
- **M486SX**：不带 FPU 的版本
- **ELAN**：AMD/Cyrix 等兼容 486 处理器

同时清理了大量过时的驱动代码：
- 旧网络驱动
- PCMCIA 驱动
- 俄罗斯 Baikal CPU 支持代码

### 你的老电脑还能用 Linux 吗？

**完全不用担心！**

| 用户类型 | 影响 | 建议 |
|:---|:---|:---|
| 2005 年以后的电脑 | ✅ 无影响 | 正常升级即可 |
| 极老的 486/586 设备 | ⚠️ 受影响 | 使用 **Linux 6.18 LTS**（长期支持） |
| 嵌入式开发者 | ⚠️ 部分影响 | 检查目标平台架构 |

Linux 6.18 LTS 将在未来数年内继续维护，专门服务于这些老旧硬件用户。这体现了 Linux 社区"**不抛弃任何人**"的理念。

## 三、其他值得关注的新特性

### 🎮 掌机玩家福音：联想 Legion Go 驱动优化

Linux 掌机生态越来越成熟了！7.1 内核专门为 **联想 Legion Go** 掌机优化了驱动程序。加上之前 Steam Deck 的良好支持，Linux 游戏掌机体验正在飞速提升。

还记得我们之前聊过的 Proton 11 吗？内核 + 运行时层的双重优化，Linux 游戏体验真的在变好。

### 🖥️ 显卡支持：Intel Xe3P 和 Nova Lake P

- **Intel Xe3P_LPG**：下一代核显的初步支持
- **Nova Lake P**：Intel 2026 年新处理器的 GPU 部分
- **NVIDIA GA100**：Nouveau 开源驱动增加了对 A100 加速器的支持

### 📱 12 款全新 SoC 支持

ARM 生态继续扩展，新增 12 款片上系统（SoC）支持，涵盖嵌入式、物联网和移动设备领域。

### 📊 代码库规模：接近 4000 万行

尽管不断清理老旧代码，Linux 内核 Git 源码树已接近 **4000 万行**大关。这个数字背后是全球数千名开发者的持续贡献。

## 四、时间线：什么时候能用上？

```
2026年4月27日  ← 现在    7.1-rc1 发布（合并窗口关闭）
     ↓
2026年5月       ← 即将到来   rc2~rc7 测试迭代
     ↓
2026年6月中下旬 ← 预计      7.1 正式版发布
     ↓
2026年Q3-Q4              Ubuntu 26.04.1 可能搭载 7.1 内核
```

### Ubuntu 用户怎么体验？

**方案一：等待官方升级（推荐）**
Ubuntu 26.04 LTS 当前搭载 Linux 7.0 内核。按惯例，**26.04.1 点版本**（预计今年 Q3 或 Q4）可能会升级到 7.1.x 内核系列。这是最稳妥的方式，LTS 用户无需折腾。

**方案二：使用 HWE 内核**
Ubuntu 提供 Hardware Enablement (HWE) 内核栈，允许 LTS 用户安装更新的内核版本：
```bash
sudo apt install linux-generic-hwe-26.04
```
等 7.1 正式版发布后，HWE 内核会很快跟进。

**方案三：手动编译（极客专属）**
```bash
# 从 kernel.org 获取源码
wget https://cdn.kernel.org/pub/linux/kernel/v7.x/linux-7.1-rc1.tar.xz
tar xf linux-7.1-rc1.tar.xz
cd linux-7.1-rc1
# 编译安装（需要较长时间和磁盘空间）
make localmodconfig
make -j$(nproc)
sudo make modules_install install
```
> ⚠️ **风险提醒**：自行编译内核可能导致系统无法启动，请务必先做好备份！

## 五、总结：为什么这次更新很重要？

| 更新维度 | 意义 | 影响范围 |
|:---|:---|:---|
| **NTFS 重写** | 解决 20+ 年的跨文件系统痛点 | WSL用户、双系统用户、NAS管理员 |
| **i486 退役** | 减少维护负担，聚焦现代硬件 | 仅影响极老设备（15年+） |
| **掌机驱动** | Linux游戏生态持续完善 | Steam Deck / Legion Go 玩家 |
| **新硬件支持** | 保持前沿硬件兼容性 | 新购机用户、开发者 |

Linux 7.1 的核心主题是**"现代化"**——拥抱新的文件系统技术，告别古老的硬件架构，同时持续扩展对新设备的支持。这正是 Linux 能够历经 30+ 年依然保持活力的原因。

**对于 Ubuntu 26.04 用户来说**，当前享受的是 Linux 7.0 + GNOME 50 的强大组合。而 7.1 的这些改进，将在未来的点版本中稳步落地。不用急，好东西值得等待。
