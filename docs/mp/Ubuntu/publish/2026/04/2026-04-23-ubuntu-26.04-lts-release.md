## 🔥 十大核心更新详解

### 1️⃣ Linux 内核 7.0——历史性版本跳跃

这是自 2019 年 Linux 5.0 发布以来的**首次大版本号跳跃**，意义非凡。

**核心亮点：**

| 特性领域 | 具体内容 |
|:---|:---|
| **Rust 集成** | Rust 正式成为内核核心语言，驱动和模块的内存安全性大幅提升 |
| **调度器优化** | 全新调度器针对 Intel 大小核架构专门优化，多核性能显著提升 |
| **文件系统进化** | XFS 文件系统新增自我修复能力，数据安全更有保障 |
| **硬件支持** | 原生支持 AMD ROCm、高通 Snapdragon X2 Elite、Intel Nova Lake |
| **游戏性能** | PREEMPT_LAZY 延迟抢占，游戏帧率提升可达 40% |

> 💡 **对普通用户的实际影响**：开机更快、应用响应更流畅、游戏帧率更高。《黑神话：悟空》实测帧率从 52fps 提升至 73fps！

### 2️⃣ sudo-rs 替代传统 sudo

你可能没注意到 `sudo` 命令的变化，但这背后意义重大。

```
# 用法完全一致，无需学习成本
sudo apt update
sudo -i
sudo nano /etc/hosts
```

**为什么换掉用了几十年的 sudo？**

- **安全加固**：传统 sudo 用 C 语言编写，存在历史遗留的内存溢出风险
- **Rust 重写**：sudo-rs 使用 Rust 编写，从根本上消除内存安全问题
- **完全兼容**：`sudoers` 配置文件格式不变，用户使用习惯零改变
- **性能相当**：日常使用几乎无感知差异

### 3️⃣ 纯 Wayland 环境——彻底告别 X11

这是 Ubuntu 桌面历史上最激进的一次显示服务器迁移。

**变化一览：**

- ✅ GDM 登录界面不再提供 X11 会话选项
- ✅ 默认仅提供 Wayland 会话
- ✅ NVIDIA 显卡在 Wayland 下大幅优化（黑屏问题修复、功耗改善）
- ⚠️ 旧版 X11 应用需通过 Xwayland 兼容层运行

**Xwayland 兼容性说明：**

```bash
# 检查当前是否运行在 Wayland
echo $XDG_SESSION_TYPE
# 输出 "wayland" 表示正在使用 Wayland

# 安装 Xwayland（通常已预装）
sudo apt install xwayland
```

> ⚠️ **注意**：如果你依赖纯 X11 的专业软件（如某些老版本的 CAD/EDA 工具），建议先在虚拟机中测试兼容性。

### 4️⃣ GNOME 50 桌面环境

GNOME 50 是本次桌面体验升级的核心。

**主要改进：**

- 🎨 动画效果更流畅，完全适配 Wayland 渲染管线
- 🔒 设置应用新增**"安全中心"**，整合加密、更新和隐私功能
- 📁 文件管理器 Nautilus 加载速度优化，网络共享性能提升
- 🔔 通知中心重新设计，信息层次更清晰
- 🖥 多显示器支持增强，外接显示器热插拔更稳定

### 5️⃣ 默认终端更换为 Ptyxis

用了多年的 GNOME Terminal 说再见，迎来全新终端模拟器。

**Ptyxis 核心特性：**

- 基于 GTK4 构建，原生现代化 UI
- 支持**模糊背景效果**（毛玻璃风格）
- 支持连字字体（编程字体渲染更美观）
- 标签页固定、会话保存功能
- 一键进入 Podman 容器（容器开发者福音）

```bash
# 如果还是想用 GNOME Terminal
sudo apt install gnome-terminal
```

### 6️⃣ 默认视频播放器更换为 Showtime

使用了十余年的 Totem 视频播放器被 Showtime 取代。

- 支持 AV1 / HEVC / VP9 现代编解码
- 界面全面现代化
- 适配沙箱环境运行
- 更好的字幕支持和播放控制

### 7️⃣ 默认启用后量子密码学

这是面向未来的安全投资。

**技术细节：**

- OpenSSH / OpenSSL 默认开启后量子加密算法 **MLKEM-768**
- 采用混合加密方案（传统椭圆曲线 + 后量子算法）
- 抵御未来量子计算机的破解风险
- 对用户透明，无需任何配置

```bash
# 验证后量子密码已启用
ssh -Q kex-algorithms | grep mlkem
```

### 8️⃣ 编译基线升级至 x86-64-v3

这是争议最大但也最必要的一项决定。

**影响分析：**

| 方面 | 详情 |
|:---|:---|
| **要求指令集** | AVX2、FMA、BMI2 |
| **性能提升** | 压缩/加密/多媒体性能 ↑5%–15% |
| **不再支持** | 2013年前 Intel CPU、2017年前 AMD CPU |
| **典型受影响** | Intel 3代及以前 Core、AMD FX 系列 |

> 💡 **如何判断你的 CPU 是否兼容？** 查看 CPU 是否支持 AVX2 指令集即可。近 10 年内购买的电脑基本都支持。

### 9️⃣ TPM 全盘加密图形化管理

安全不再是命令行高手的专利。

**新功能：**

- 在"安全中心"内直接管理 TPM 加密
- 可视化修改 PIN、轮换密钥
- 全程 GUI 操作，无需敲命令
- 加密性能损耗进一步降低

### 🔟 软件管理中心统一

告别混乱的多工具时代。

- **App Center** 成为唯一推荐 GUI 管理工具
- 统一管理 deb、Snap、Flatpak、PPA
- `software-properties-gtk` 不再默认安装
- `apt`、`add-apt-repository` 等命令行工具**完全不受影响**

## 🚀 升级指南：四种方式任你选

### 方式一：从 24.04 LTS 直接升级（推荐）

```bash
# 1. 更新当前系统
sudo apt update && sudo apt full-upgrade -y

# 2. 安装 update-manager-core
sudo apt install update-manager-core

# 3. 执行发行版升级
sudo do-releaseupgrade -d

# 4. 按照提示完成升级（大约需要30-60分钟）
```

### 方式二：全新安装（最干净）

1. 访问 [ubuntu.com/download](https://cn.ubuntu.com/download/desktop)
2. 下载 Ubuntu 26.04 LTS ISO 镜像
3. 使用 Rufus/Ventoy 制作启动U盘
4. 从U盘引导安装

### 方式三：虚拟机体验（零风险）

```bash
# 使用 Virt-Manager 安装
sudo apt install virt-manager qemu-system-x86
# 导入ISO镜像创建新虚拟机
```

### 方式四：WSL 子系统体验

```powershell
# PowerShell 中执行
wsl --install -d Ubuntu-26.04-LTS
```

## ⚠️ 升级前必读：避坑清单

### ✅ 升级前准备

1. **备份数据**！（最重要！）
   ```bash
   # 使用 Deja Dup 备份 home 目录
   deja-dup --backup
   ```

2. 记录已安装的 PPA：
   ```bash
   find /etc/apt/sources.list.d -type f -name "*.list"
   ```

3. 清理无用软件包：
   ```bash
   sudo apt autoremove --purge
   ```

4. 确认磁盘空间至少剩余 15GB+

### ⚠️ 已知注意事项

| 问题 | 解决方案 |
|:---|:---|
| 老电脑无法安装（CPU不支持AVX2） | 继续使用 24.04 LTS 或选择轻量版 |
| NVIDIA 旧显卡驱动问题 | 等待 NVIDIA 官方适配或使用开源 Nouveau 驱动 |
| 部分 Snap 应用启动慢 | 首次启动需要重新编译缓存，后续正常 |
| Xwayland 兼容性问题 | 大部分应用可正常运行，个别老旧软件可能异常 |

### 👍 推荐升级人群

- 桌面用户想体验最新功能
- 开发者需要新版内核和开发工具
- 硬件较新的用户（2017年后购买）
- 安全敏感用户（后量子加密、sudo-rs）

### 👎 暂缓升级人群

- 生产环境服务器（建议等 8 月的 26.04.1）
- 老旧硬件用户（不支持 x86-64-v3）
- 依赖特定 X11 应用的专业用户
- 追求极致稳定的保守型用户
