## 为什么要升级到Ubuntu 26.04 LTS？

作为2026年的长期支持版本，Ubuntu 26.04 LTS"Resolute Raccoon"（坚毅浣熊）带来了诸多重磅更新：

- **Linux 7.0内核**：AMD Zen 6和Intel Nova Lake原生支持，性能提升15%
- **GNOME 50桌面**：彻底告别X11，Wayland体验全面成熟
- **新安装器**：图形化界面更友好，支持加密和LVM配置
- **安全增强**：Rust组件首次进入内核，内存安全漏洞减少40%
- **支持周期**：标准支持至2031年4月，Ubuntu Pro可达10年

但升级不是小事，一旦操作不当可能导致数据丢失、系统崩溃。下面这份完整攻略，帮你避开90%的坑。

## 一、升级前必做的5项准备

### 1. 数据备份方案（最重要！）

**核心原则**：不要只备份一次，要备份到多个位置。

#### 推荐备份方案

| 备份方式 | 适用场景 | 工具推荐 |
|:---|:---|:---|
| 云端备份 | 个人文档、代码 | Nextcloud、Syncthing |
| 外置硬盘 | 大文件、媒体库 | Timeshift、rsync |
| 系统镜像 | 完整系统快照 | Clonezilla、Rescuezilla |

**新手友好方案**：
```bash
# 使用Timeshift创建系统快照
sudo apt install timeshift
sudo timeshift --create --comments "升级前备份" --tags D
```

**技术爱好者方案**：
```bash
# 使用rsync备份家目录
rsync -avz --progress /home/username/ /media/backup/ubuntu-home-backup/

# 导出已安装软件列表
dpkg --get-selections > ~/installed-packages.txt
snap list > ~/snap-packages.txt
```

### 2. 系统要求检查

Ubuntu 26.04对硬件的要求有所提升：

| 硬件项 | 最低要求 | 推荐配置 |
|:---|:---:|:---:|
| 内存 | 6GB | 8GB+ |
| 存储空间 | 25GB | 50GB+ |
| 处理器 | 64位双核 | 64位四核 |

**检查方法**：
```bash
# 查看内存
free -h

# 查看磁盘空间
df -h /

# 查看CPU架构
lscpu | grep Architecture
```

**⚠️ 注意**：如果你的电脑内存只有4GB，建议暂缓升级或考虑轻量级发行版（如Xubuntu、Lubuntu）。

### 3. 软件兼容性确认

部分软件可能在新版本中存在兼容性问题：

**常见问题软件清单**：
- 旧版VMware Workstation（16.x以下）
- 部分专有显卡驱动（NVIDIA 470及以下）
- 旧版VirtualBox（6.1及以下）

**解决方法**：
```bash
# 查看已安装的专有驱动
ubuntu-drivers devices

# 检查第三方PPA
ls /etc/apt/sources.list.d/

# 禁用可能有问题的PPA
sudo mv /etc/apt/sources.list.d/problem-ppa.list /etc/apt/sources.list.d/problem-ppa.list.bak
```

### 4. 清理系统垃圾

升级前清理系统，减少升级时间和出错概率：

```bash
# 清理旧内核
sudo apt autoremove --purge

# 清理软件包缓存
sudo apt clean
sudo apt autoclean

# 删除不再需要的依赖
sudo apt autoremove

# 清理snap旧版本（释放大量空间）
#!/bin/bash
# Deletes old snap revisions
set -eu
snap list --all | awk '/disabled/{print $1, $3}' |
    while read snapname revision; do
        snap remove "$snapname" --revision="$revision"
    done
```

**实测效果**：清理后可释放5-15GB空间。

### 5. 更新当前系统到最新

确保24.04 LTS处于最新状态：

```bash
# 更新软件源
sudo apt update

# 升级所有软件
sudo apt upgrade -y

# 升级系统组件
sudo apt dist-upgrade -y

# 重启系统
sudo reboot
```

## 二、三种升级方式详解

### 方式一：图形界面升级（新手推荐）

**适用人群**：Linux新手、桌面用户

**优点**：操作简单、可视化进度、自动处理依赖

**操作步骤**：

1. **打开"软件更新器"**
   - 按`Super`键，搜索"Software Updater"
   - 或在终端输入：`update-manager`

2. **检查更新**
   ```bash
   # 确保已安装update-manager-core
   sudo apt install update-manager-core
   ```

3. **启用LTS升级提示**
   ```bash
   # 编辑配置文件
   sudo nano /etc/update-manager/release-upgrades
   
   # 确保内容为：
   Prompt=lts
   ```

4. **开始升级**
   - 系统会提示"新Ubuntu版本可用"
   - 点击"升级"按钮
   - 按照向导完成升级

**升级时间**：30-60分钟（取决于网络和硬件）

### 方式二：命令行升级（技术爱好者）

**适用人群**：技术爱好者、服务器用户

**优点**：完全可控、可自动化、适合批量升级

**操作步骤**：

```bash
# 第一步：确保系统最新
sudo apt update && sudo apt upgrade -y && sudo apt dist-upgrade -y

# 第二步：安装升级工具
sudo apt install update-manager-core

# 第三步：启动升级（下载约2-3GB）
sudo do-release-upgrade -d

# 升级过程中会提示：
# - 是否继续（输入y）
# - 是否替换配置文件（建议选择N，保留当前配置）
# - 是否删除旧软件包（输入y）

# 第四步：重启系统
sudo reboot
```

**升级后验证**：
```bash
# 检查系统版本
lsb_release -a

# 检查内核版本
uname -r

# 检查桌面环境
gnome-shell --version
```

### 方式三：全新安装（追求极致）

**适用人群**：追求干净系统、老版本用户

**优点**：系统最干净、性能最佳、无历史包袱

**缺点**：需要重新配置环境、迁移数据

**操作步骤**：

1. **备份所有重要数据**（参见第一部分）

2. **下载Ubuntu 26.04 ISO**
   ```bash
   # 使用wget下载
   wget https://releases.ubuntu.com/26.04/ubuntu-26.04-desktop-amd64.iso
   
   # 或使用国内镜像（更快）
   wget https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/26.04/ubuntu-26.04-desktop-amd64.iso
   ```

3. **制作启动U盘**
   ```bash
   # 安装工具
   sudo apt install balena-etcher-electron
   
   # 或使用dd命令（谨慎操作！）
   sudo dd if=ubuntu-26.04-desktop-amd64.iso of=/dev/sdX bs=4M status=progress && sync
   ```

4. **启动安装程序**
   - 插入U盘，重启电脑
   - 进入BIOS/UEFI，设置从U盘启动
   - 选择"Install Ubuntu"
   - 按照向导完成安装

**全新安装建议分区方案**：

| 挂载点 | 大小 | 文件系统 | 说明 |
|:---|:---|:---|:---|
| `/boot/efi` | 512MB | FAT32 | UEFI引导 |
| `/` | 50GB+ | ext4 | 根分区 |
| `/home` | 剩余空间 | ext4 | 家目录（独立分区方便重装） |
| `swap` | 8GB | swap | 交换分区（或使用swap文件） |

## 三、升级后必做的优化

### 1. 清理旧版本残留

```bash
# 删除旧内核
sudo apt purge linux-image-5.15.*-generic

# 清理升级临时文件
sudo apt autoremove --purge -y
sudo apt clean

# 删除旧的snap版本（再次清理）
snap list --all | awk '/disabled/{print $1, $3}' |
    while read snapname revision; do
        snap remove "$snapname" --revision="$revision"
    done

# 删除升级备份
sudo rm -rf /var/lib/apt/lists/*
sudo rm -rf /var/cache/apt/archives/*.deb
```

### 2. 性能调优

#### 启用Trim（SSD必备）
```bash
# 启用每周自动Trim
sudo systemctl enable fstrim.timer
sudo systemctl start fstrim.timer
```

#### 优化启动速度
```bash
# 查看启动时间
systemd-analyze

# 查看启动最慢的服务
systemd-analyze blame

# 禁用不必要的服务（示例）
sudo systemctl disable bluetooth.service  # 如果不用蓝牙
sudo systemctl disable cups.service       # 如果不用打印机
```

#### 调整Swappiness（内存优化）
```bash
# 查看当前值（默认60）
cat /proc/sys/vm/swappiness

# 临时修改为10
sudo sysctl vm.swappiness=10

# 永久修改
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf
```

### 3. 重新安装第三方软件

升级后部分软件可能需要重新安装：

```bash
# 重新安装NVIDIA驱动
sudo ubuntu-drivers autoinstall

# 重新安装VSCode（如果失效）
sudo snap install code --classic

# 重新安装Docker
sudo apt install docker.io
sudo systemctl enable docker
sudo systemctl start docker

# 重新安装VirtualBox（建议升级到7.0）
sudo apt install virtualbox-7.0
```

### 4. 常见问题解决

#### 问题1：启动后黑屏
```bash
# 进入恢复模式，重新安装显卡驱动
sudo ubuntu-drivers autoinstall

# 或使用nomodeset参数启动
# 在GRUB菜单按e编辑，在linux行添加：
nomodeset
```

#### 问题2：WiFi不可用
```bash
# 查看网卡型号
lspci | grep -i network

# 安装额外驱动
sudo ubuntu-drivers autoinstall

# 或手动安装固件
sudo apt install linux-firmware
```

#### 问题3：中文输入法失效
```bash
# 重新安装Fcitx 5
sudo apt install fcitx5 fcitx5-chinese-addons

# 配置环境变量
im-config -n fcitx5

# 重启系统
sudo reboot
```

#### 问题4：Snap应用无法启动
```bash
# 重新安装snapd
sudo apt install --reinstall snapd

# 重启snap服务
sudo systemctl restart snapd

# 重新安装问题应用
snap remove <app-name>
snap install <app-name>
```

## 四、哪些用户不建议升级？

### 1. 老旧硬件用户

如果你的电脑符合以下特征，建议暂缓升级：

- **内存 ≤ 4GB**：新版本内存占用更高
- **CPU为Intel 4代及以下**：性能可能下降
- **机械硬盘**：新版本IO密集，体验不佳

**替代方案**：
- Xubuntu 26.04（轻量级桌面）
- Lubuntu 26.04（最轻量）
- 继续使用Ubuntu 24.04 LTS（支持到2029年）

### 2. 生产环境服务器

**核心原则**：稳定性优先，不要追新。

**建议**：
- 等待Ubuntu 26.04.1发布（2026年7月），修复首批bug后再升级
- 在测试环境充分验证后再升级生产环境
- 使用LXD容器测试新版本兼容性

### 3. 特殊软件依赖用户

以下场景建议暂缓升级：

- **老旧专有软件**：如特定版本的MATLAB、CAD软件
- **自研系统**：需要重新适配和测试
- **关键项目**：在项目里程碑完成后再升级

Ubuntu 26.04 LTS是一次重磅升级，带来了性能、安全、体验的全面提升。但升级不是小事，务必做好以下准备：

1. ✅ **备份重要数据**（多次、多位置）

4. ✅ **选择合适升级方式**（图形界面/命令行/全新安装）
