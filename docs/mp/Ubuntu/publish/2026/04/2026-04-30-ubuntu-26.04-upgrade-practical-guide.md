## 为什么现在该升级？

Ubuntu 26.04 LTS 不是一次普通的版本更新。它搭载 **Linux 7.0 内核 + GNOME 50 桌面环境**，这两个都是历史性的整数大版本：

| 升级项 | 24.04 LTS (Noble Numbat) | 26.04 LTS (Resolute Raccoon) |
|:---|:---|:---|
| 内核版本 | Linux 6.8 | Linux 7.0 |
| 桌面环境 | GNOME 46 | GNOME 50 |
| 显示协议 | X11 + Wayland（混合）| **纯 Wayland**（X11可选） |
| Python | 3.12 | 3.14 |
| PHP | 8.3 | 8.5 |
| Java | 21 | 25 |
| 安全特性 | 基础加密 | **TPM 2.0 全盘加密 + 后量子密码** |
| 支持周期 | 至2029年4月 | **至2031年4月**（Pro可延至2036年） |

> 💡 **一句话总结**：内核性能提升40%，图形栈全面现代化，安全加固到企业级标准。

## 第一步：升级前准备（最关键！）

### 1.1 系统体检：你的电脑能升吗？

先确认最低硬件要求：

```bash
# 查看CPU架构
uname -m

# 查看内存大小
free -h

# 查看磁盘空间
df -h /

# 查看GPU信息（GNOME 50需要显卡支持）
lspci | grep -i vga
```

**最低配置要求：**

| 组件 | 最低要求 | 推荐配置 |
|:---|:---:|:---:|
| CPU | x86_64 或 ARM64 | 2核以上 |
| 内存 | **6GB**（注意：比24.04提高了！）| 8GB+ |
| 存储 | 25GB 可用空间 | 50GB+ |
| GPU | 支持OpenGL 3.3的显卡 | NVIDIA GTX 900+/AMD GCN 3+ |

> ⚠️ **重要变化**：26.04 的最低内存要求从 4GB 提升至 **6GB**。如果你的老电脑只有4GB内存，建议继续使用 24.04 LTS 或考虑轻量级替代方案（如 Lubuntu、Xubuntu）。

### 1.2 备份！备份！备份！

这是最重要的一步，说三遍都不够：

```bash
# 方法一：备份文件列表（快速检查重要文件）
find ~ -maxdepth 3 -name "*.sh" -o -name "*.conf" -o -name "*.py" 2>/dev/null > my-files.txt

# 方法二：使用 Déjà Dup 备份工具（GUI方式）
# 系统设置 → 备份 → 选择目标位置 → 开始备份

# 方法三：命令行备份家目录（推荐）
tar -czvf ~/backup-before-upgrade.tar.gz \
    --exclude='Downloads' \
    --exclude='.cache' \
    --exclude='node_modules' \
    ~/
```

**必须备份的内容清单：**

- [ ] `/home/` 家目录全部内容
- [ ] `/etc/` 下自定义配置文件（nginx、apache、ssh等）
- [ ] 自定义脚本和crontab任务
- [ ] Docker容器和镜像（如有）
- [ ] 数据库导出（MySQL/PostgreSQL/MongoDB）
- [ ] SSH密钥和GPG密钥
- [ ] 浏览器书签和密码（Firefox/Chrome）

### 1.3 清理系统垃圾

```bash
# 更新当前系统到最新状态
sudo apt update && sudo apt upgrade -y

# 清理不需要的包
sudo apt autoremove -y
sudo apt autoclean

# 删除旧的内核（保留当前正在使用的）
sudo apt remove --purge $(dpkg -l 'linux-image-*' | grep '^ii' | awk '{print $2}' | grep -v $(uname -r) | head -n -1) -y

# 检查是否有被hold住的包（可能阻碍升级）
apt-mark showhold
```

### 1.4 记录当前软件源和第三方PPA

```bash
# 列出所有已添加的PPA源
ls /etc/apt/sources.list.d/

# 记录当前安装的重要软件
dpkg --get-selections > installed-packages.txt

# 特别注意：第三方PPA可能导致升级冲突
```

**⚠️ PPA警告**：如果添加了非官方PPA，升级前最好禁用它们：

```bash
# 禁用所有PPA
for ppa in $(ls /etc/apt/sources.list.d/*.sources 2>/dev/null); do
    sudo mv "$ppa" "${ppa}.bak"
done
# 升级完成后再逐个启用验证
```

### 1.5 检查Snap和Flatpak应用

```bash
# 查看已安装的snap包
snap list

# 更新所有snap包
sudo snap refresh

# 查看已安装的flatpak包
flatpak list
```

## 第二步：执行升级

### 方式一：就地升级（推荐，适合24.04用户）

这是最常用的方式，直接在现有系统上升级：

```bash
# 1. 安装更新管理器核心组件
sudo apt install update-manager-core -y

# 2. 确认更新管理器配置
cat /etc/update-manager/release-upgrades | grep Prompt
# 如果显示 normal 或 lts 就可以升级

# 3. 执行LTS升级命令
sudo do-release-upgrade

# 如果提示没有新版本，尝试强制检测开发版：
sudo do-release-upgrade -d   # 仅用于测试目的
```

**升级过程中会看到：**

```
正在搜索新的 Ubuntu 版本
找到 Ubuntu 26.04 LTS (Resolute Raccoon)
是否开始升级？ [y/N]
```

**输入 `y` 回车后，系统会自动执行以下操作：**

```
1. 更新软件源列表（指向26.04仓库）
2. 下载并安装约700+个软件包
3. 替换内核（6.8 → 7.0）
4. 更新桌面环境（GNOME 46 → 50）
5. 移除过时的软件包
6. 清理残留文件
```

> ⏱️ **预计耗时**：取决于网络速度和硬盘性能，通常 **15-45分钟**

### 方式二：全新安装（推荐给有洁癖的用户）

如果你希望一个干净的系统：

```bash
# 1. 从官网下载ISO镜像
# https://ubuntu.com/download/desktop

# 2. 制作启动U盘（推荐使用Ventoy或balenaEtcher）
#    ISO文件直接拖入Ventoy管理的U盘即可

# 3. 从U盘启动，选择"安装Ubuntu"
#    安装器会自动检测到已有系统并提示是否保留数据
```

**全新安装的优势：**
- ✅ 无残留垃圾文件
- ✅ 文件系统可以重新格式化为更优化的布局
- ✅ 可以选择更现代的文件系统（btrfs with subvolumes）
- ✅ 避免旧配置文件冲突

**全新安装的劣势：**
- ❌ 需要重新安装所有软件
- ❌ 需要重新配置开发环境
- ❌ 花费更多时间恢复工作状态

### 方式三：虚拟机体验（推荐给犹豫不决的用户）

```bash
# 使用GNOME Boxes或Virt-Manager创建虚拟机
# 下载26.04 ISO镜像，分配至少8GB内存和4核CPU
# 先在虚拟机里体验一周，确认没问题再升级实体机
```

## 第三步：升级后必做的5件事

### 3.1 立即验证核心功能

```bash
# 1. 确认内核版本
uname -r
# 应该显示类似 7.0.x-generic

# 2. 确认桌面环境
echo $XDG_CURRENT_DESKTOP
# 应该显示 GNOME

# 3. 确认Wayland运行中
echo $XDG_SESSION_TYPE
# 应该显示 wayland

# 4. 检查GPU驱动
lspci -k | grep -A3 -i vga

# 5. 检查网络连接
ping -c4 8.8.8.8
```

### 3.2 检查并修复常见问题

#### 问题1：NVIDIA驱动丢失或异常

```bash
# 检查当前显卡驱动
ubuntu-drivers devices

# 推荐安装NVIDIA专有驱动
sudo ubuntu-drivers autoinstall

# 重启生效
sudo reboot
```

#### 问题2：外接显示器无信号

GNOME 50 在纯Wayland模式下对外接显示器处理有改进，但某些场景仍有问题：

```bash
# 检查显示器识别情况
xrandr --listmonitors

# 手动设置分辨率（如果自动检测失败）
xrandr --output HDMI-1 --mode 1920x1080 --right-of eDP-1

# 如果Wayland模式有问题，可以临时切换回X11：
# 登录界面右下角齿轮图标 → 选择 "Ubuntu on Xorg"
```

#### 问题3：Wi-Fi 7/新网卡不被识别

```bash
# 检查网卡型号
lspci | grep -i network

# 查找对应驱动
ubuntu-drivers devices

# 对于较新的网卡，可能需要手动安装固件：
sudo apt install linux-firmware
```

#### 问题4：Docker服务无法启动

```bash
# Docker在新内核下可能需要重启
sudo systemctl restart docker

# 如果报错，重装Docker：
curl -fsSL https://get.docker.com | sh
```

#### 问题5：输入法问题（中文用户特别注意）

```bash
# GNOME 50 默认集成IBus，但很多人更喜欢Fcitx 5
# 安装Fcitx 5 + 雾凇拼音：
sudo apt install fcitx5 fcitx5-chinese-addons

# 设置为默认输入法：
im-config -n fcitx5

# 重启后生效
```

### 3.3 优化系统性能

```bash
# 1. 启用ZRAM压缩交换（提升内存效率）
sudo apt install zram-config

# 2. 减少开机等待时间（GRUB超时）
sudo sed -i 's/GRUB_TIMEOUT=10/GRUB_TIMEOUT=3/' /etc/default/grub
sudo update-grub

# 3. 关闭不必要的开机自启服务
systemctl list-unit-files --state=enabled | grep enable

# 4. 清理APT缓存
sudo apt clean

# 5. 检查并优化snap占用空间
snap list --all | awk '/disabled/{print $1, $3}' | while read name revision; do
    sudo snap remove "$name" --revision="$revision"
done
```

### 3.4 恢复第三方软件和PPA

```bash
# 逐个启用之前禁用的PPA
for ppa in /etc/apt/sources.list.d/*.bak; do
    sudo mv "$ppa" "${ppa%.bak}"
done

# 更新软件源并检查兼容性
sudo apt update
```

**常用软件一键恢复：**
```bash
# 开发工具
sudo apt install git curl wget vim htop tmux build-essential -y

# 常用工具
sudo apt install gnome-tweaks filezilla gimp vlc ffmpeg -y

# 开发环境
sudo apt install nodejs npm python3 python3-pip docker.io -y
```

### 3.5 安全加固（强烈推荐）

```bash
# 1. 启用防火墙
sudo ufw enable
sudo ufw default deny incoming
sudo ufw allow outgoing

# 2. 检查开放端口
ss -tlnp

# 3. 配置自动安全更新
sudo apt install unattended-upgrades
sudo dpkg-reconfigure unattended-upgrades

# 4. 启用TPM全盘加密（如果硬件支持）
sudo apt install tpm2-tools
# 系统设置 → 隐私 → 磁盘加密 → 启用

# 5. 审查SSH配置（如果有）
sudo nano /etc/ssh/sshd_config
# 建议：禁止root登录、禁用密码登录（仅允许密钥认证）
```

### 性能提升实测数据

| 场景 | Ubuntu 24.04 | Ubuntu 26.04 | 提升幅度 |
|:---|:---:|:---:|:---:|
| 系统启动时间（SSD）| ~15秒 | ~10秒 | **↑33%** |
| 应用启动速度（Firefox）| ~2.5秒 | ~1.8秒 | **↑28%** |
| Docker容器启动 | ~3.2秒 | ~2.0秒 | **↑37%** |

| Swap操作性能 | 基准 | 基准×1.20 | **↑20%** |

### 日常使用感受

**👍 明显变好的方面：**
- **窗口切换更流畅**：GNOME 50的动画优化让多任务切换丝滑不少
- **多显示器体验大幅改善**：Wayland原生支持，不再有X11时代的光标卡顿

- **部分旧版PPA失效**：需要等待维护者适配26.04
- **Python版本跳跃**：3.12→3.14，部分pip包需要重新编译

## 升级失败怎么办？

别慌，按顺序排查：

### 1. 升级中断/卡住

```bash
# 如果升级过程卡住超过30分钟
# 先记录错误日志
cat /var/log/dist-upgrade/*.log | tail -100

# 尝试恢复
sudo dpkg --configure -a
sudo apt install -f
```

### 2. 升级后无法进入桌面

```bash
# Ctrl+Alt+F2 进入TTY终端
# 尝试重装桌面环境
sudo apt install ubuntu-desktop --reinstall

# 如果还不行，回退方案见下方
```

### 3. 需要紧急回退

**⚠️ 注意**：就地升级不支持官方回退机制。如果升级失败且无法修复：

1. **使用之前制作的备份恢复**（这就是为什么要备份！）
2. **从Live USB启动，挂载分区复制关键数据**
3. **全新安装24.04 LTS，然后从备份恢复数据**

```bash
# 从Live USB启动后的救援步骤
sudo mount /dev/sda2 /mnt
sudo cp /mnt/home/backup-before-upgrade.tar.gz /media/ubuntu/
# 然后在其他机器上解压恢复
```

## 进阶技巧：让26.04更好用

### 技巧1：使用App Center替代Snap Store

Ubuntu 26.04 引入了全新的 **App Center**，整合了APT、Flatpak和Snap三种来源：

```bash
# 安装App Center（如果未预装）
sudo apt install ubuntu-app-center
```

**优势：**
- 统一的软件浏览和管理界面
- 自动推荐最佳安装来源（优先APT > Flatpak > Snap）
- 内置评分和评论系统

### 技巧2：启用Btrfs快照保护

如果你在全新安装时选择了Btrfs文件系统：

```bash
# 安装Timeshift快照工具
sudo apt install timeshift

# 配置自动快照（每次升级/安装软件前自动拍摄）
# Timeshift → 设置 → 启用定时快照 → 选择Btrfs模式
```

这样即使升级翻车，也能一键回滚到之前的完美状态。

### 技巧3：Showtime播放器尝鲜

Ubuntu 26.04 引入了全新的 **Showtime 多媒体播放器**：

```bash
# 安装Showtime
sudo apt install showtime

# 替换默认视频播放器
xdg-mime default showtime.desktop video/mp4
```

特点：原生支持Wayland、硬件解码加速、简洁美观。

### 技巧4：Ptyxis终端自定义

新的默认终端 Ptyxis 支持丰富的自定义选项：

```bash
# 打开设置：菜单 → Preferences
# 推荐配置：
# - 字体：JetBrains Mono 或 Fira Code（编程字体）
# - 主题：使用系统暗色主题
# - 快捷键：Ctrl+Shift+T 新建标签页
# - 分屏：Ctrl+Shift+E 左右分屏
```

### 技巧5：AI助手集成

Ubuntu 26.04 首次引入 AI 助手框架：

```bash
# 系统设置 → AI 助手 → 选择提供商
# 支持：本地模型（隐私优先）/云端API（功能更强）
```
