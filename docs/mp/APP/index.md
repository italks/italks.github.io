## 1. 品牌识别系统 (Visual Identity)

为了建立“专业、可靠、懂生意”的专家形象，所有对外输出的内容（文章、图片、视频）需遵循以下视觉规范。

### 🎨 品牌色板 (Color Palette)

我们采用**“深蓝商务风”**作为主基调，辅以高饱和度的暖色作为点缀，体现科技感与商业活力的平衡。

| 色彩名称 | 色值 (HEX) | 使用场景 | 心理暗示 |
| :--- | :--- | :--- | :--- |
| **深海蓝 (主色)** | `#1E3C72` | 封面背景、标题栏、品牌Logo | 沉稳、专业、信任、科技 |
| **科技蓝 (辅色)** | `#2A5298` | 渐变色结束端、次级按钮 | 现代、连接、未来 |
| **警示黄 (强调)** | `#FFCB05` | 关键数据、避坑指南、价格标签 | 注意、高亮、金钱、警示 |
| **活力红 (点缀)** | `#FF6B6B` | 否定/错误图示、促销/钩子 | 紧迫感、热点、行动 |
| **极简白 (背景)** | `#FFFFFF` | 正文背景、卡片底色 | 干净、清晰 |

### 🔠 字体规范 (Typography)

*   **封面/海报标题：** 推荐使用 **思源黑体 Heavy** 或 **微软雅黑 Bold**。要求笔画粗壮，有力量感。
*   **正文内容：** 微信默认字体（iOS: PingFang SC, Android: Roboto/Noto Sans）。
*   **代码片段：** 必须使用等宽字体，如 `JetBrains Mono` 或 `Consolas`，配色推荐 GitHub Dark 风格。
*   **数字/金额：** 使用 **DIN** 或 **Arial**，突出数据的严谨性。

## 2. 封面图设计规范 (Cover Design)

封面图是用户打开文章的第一动力。我们的封面必须传达**“这是一个有价值的解决方案”**。

### 📐 构图模板

1.  **左文右图结构（推荐）：**
    *   **左侧 (60%)：** 放置主标题（大号字体）+ 副标题/标签（反白或高亮色块）。
    *   **右侧 (40%)：** 放置具象化的插图（如手机模型、架构图、报表）。
    *   *适用场景：* 解决方案、案例复盘、成本分析。

2.  **居中大字结构：**
    *   **背景：** 深色品牌渐变 (`#1E3C72` -> `#2A5298`)。
    *   **中心：** 极具冲击力的文案（如“**这三个坑，别踩！**”）。
    *   *适用场景：* 避坑指南、观点输出、紧急热点。

### 🚫 设计禁忌

*   ❌ **拒绝卡通化：** 不要使用低幼的卡通人物，我们要对话的是老板和CTO。
*   ❌ **拒绝纯代码截图：** 除非是纯技术教程，否则不要直接把满屏代码当封面，会让非技术客户退缩。
*   ❌ **拒绝花哨背景：** 背景不要干扰文字阅读，保持干净。

## 3. 文章排版风格 (Article Style)

### 📝 内容结构：SCQA 模型

每一篇文章都应该像一份**咨询报告**，而不是流水账。

1.  **情境 (Situation)：** 描述客户当下的商业环境或痛点（如：流量贵、APP卡顿）。
2.  **冲突 (Complication)：** 为什么这个问题很难解决？（如：市面上外包坑多、技术选型复杂）。
3.  **问题 (Question)：** 那么，我们该怎么办？
4.  **答案 (Answer)：** 给出我们的专业方案（成本拆解、架构图、案例数据）。

### 🧩 常用组件样式

*   **引用/金句：** 使用左侧竖线引用样式。
    > 💡 **专家提示：** 所有的技术债，最终都会变成财务债。

*   **数据可视化：** 涉及成本、性能提升、用户增长的数据，必须使用**表格**或**图表**展示，不要只用文字。

*   **代码块：** 仅展示关键逻辑，行数控制在 20 行以内，必须加注释。

### 🔗 转化钩子 (Call to Action)

文章结尾必须包含统一的商务联系卡片，引导客户进行下一步行动。

**标准话术模板：**

> **您的业务也面临同样的挑战吗？**
>
> 拒绝套路，我们提供**免费**的 1 对 1 技术诊断与成本评估。
>
> 👇 **点击下方名片关注，回复“方案”，获取您的专属数字化建议。**
