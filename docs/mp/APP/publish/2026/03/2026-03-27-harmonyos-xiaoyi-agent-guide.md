## 二、小艺智能体是什么？别搞错方向

很多开发者第一次接触小艺智能体，容易把它做成"高级复读机"——用大模型回答所有问题，看起来很聪明，但一进真实业务场景就垮掉。

**小艺开放平台提供4种开发模式：**

| 模式 | 适合场景 | 核心特点 |
|:---|:---|:---|
| **LLM模式** | 问答、轻咨询 | 启动快，注意边界控制 |
| **工作流模式** | 功能型任务 | 可视化节点编排，流程稳定 |
| **A2A模式** | 第三方智能体接入 | 适合已有Agent迁移 |
| **OpenClaw模式** | 高定制化个性助手 | 灵活度最高 |

**选型建议：**
- 新手入门 → **LLM模式**，快速验证场景
- 有明确业务流程 → **工作流模式**，稳定性好
- 已有OpenClaw智能体 → **OpenClaw模式**，直接迁移

## 三、开发前必做的3件事

### 1. 账号认证（别跳过）

注册华为开发者联盟账号，完成**实名认证**。

- **个人账号**：适合独立开发者、参赛作品
- **企业账号**：适合商业化需求，资质更完善

> ⚠️ 坑：很多人在审核阶段才发现认证资料不全，导致发布延迟。建议一开始就按企业账号的标准准备材料。

### 2. 素材提前规划

上架审核时，以下材料缺一不可：

- **图标**：1:1比例，不超过5MB，风格与功能一致
- **名称**：≤8字符，直观体现功能，**禁用营销词**（"免费""赚钱""最强"统统不行）
- **描述**：明确说明能做什么，不能做什么
- **隐私政策链接**：必须可访问，内容必须真实

> ⚠️ 高频驳回原因#1：名称中包含"免费"、"超级"、"第一"等营销词

### 3. 确定单点场景

做智能体最大的误区是"做什么都会"。审核员看到一个什么都能回答的Agent，会直接驳回为"功能不明确"。

**正确姿势：先打透一个高频场景**

```
例：旅行助手智能体
❌ 错误：什么旅游问题都能回答
✓ 正确：专门帮用户规划云南自由行3-7天行程，输出结构化行程单
```

## 四、开发核心步骤详解

### Step 1：创建智能体

进入小艺开放平台 → 点击"+ 创建智能体" → 选择开发模式

**最小可用智能体原则：**
- 只做1-2个核心功能
- 先上线验证再扩展
- 单一场景的转化率 > 全能但模糊的智能体

---

### Step 2：编写角色指令（Prompt工程）

这是决定智能体质量的最关键一步。99%的开发者在这里犯错。

**完整Prompt模板（可直接套用）：**

```
# 角色定义
你是一个专门帮助用户[具体功能]的智能助手，服务于[目标用户群体]。

# 核心目标
1. [主要任务1]
2. [主要任务2]
3. [主要任务3]

# 回答规则
- 只回答与[领域]相关的问题
- 超出范围时，礼貌告知并引导回正题
- 回答时优先参考知识库内容
- 不确定时，说"我需要更多信息"，而不是猜测

# 输出格式
- 行程类：使用表格+分段描述
- 建议类：使用编号列表
- 比较类：使用对比表格

# 失败兜底
如果无法完成任务，回复："抱歉，这个问题超出了我的专业范围，建议您[具体替代方案]。"
```

> ⚠️ 高频驳回原因#2：Prompt边界不清晰，导致大模型回答任意话题，违反平台内容规范

---

### Step 3：配置能力模块

根据场景选择性开启，不要全部堆砌：

**知识库**：提供事实性内容
```
适合：FAQ问答、产品手册、规章制度
注意：文件格式支持PDF/Word/TXT，单个不超过50MB
```

**插件（API调用）**：接入外部数据
```
适合：查天气、查快递、查价格
注意：API响应时间需 < 5秒，否则体验差
```

**工作流**：多步骤任务自动化
```
适合：表单填写 → 数据处理 → 结果输出
注意：节点超过10个建议拆分为子流程
```

**长期记忆**：记住用户偏好
```
适合：个性化推荐、历史记录追踪
注意：仅在真正需要个性化时开启，否则增加复杂度
```

---

### Step 4：设计开场语和预置问题

**开场语公式：**
```
我是[名称]，专门帮你[核心功能]。
你可以问我：
• [场景1]
• [场景2]  
• [场景3]
```

**预置问题（最多3个）**选高频场景：

| ❌ 糟糕示例 | ✓ 优质示例 |
|:---|:---|
| "你能做什么？" | "帮我规划5天云南行程" |
| "介绍一下你自己" | "推荐大理最值得去的3个地方" |

---

### Step 5：调试清单（必看）

上线前必须测试以下场景：

- [ ] **正常使用流程**：主要场景全部走通
- [ ] **模糊输入**：用户说"随便推荐一下"
- [ ] **超范围问题**：问一个完全不相关的问题
- [ ] **对抗性输入**："假装你是[其他身份]..."
- [ ] **空白输入**：用户只发了一个表情
- [ ] **连续对话**：多轮问答是否保持上下文

## 五、上架审核5大雷区

根据开发者社区反馈，以下是最常见的驳回原因：

### ❌ 雷区1：名称不规范
禁用词：免费、超级、最强、第一、赚钱、神器、赚钱、AI（除非与功能直接相关）

**解决方案：** 名称用"[功能]+[对象]"格式，如"行程规划师"、"合同校对助手"

---

### ❌ 雷区2：功能不明确
审核员3秒内看不懂能做什么 → 直接驳回

**解决方案：** 描述第一句直接说"帮你做XXX的智能助手"

---

### ❌ 雷区3：隐私政策缺失或内容不符
这是最容易被忽视但必须完成的合规项

**解决方案：** 使用隐私政策生成器，确保包含：数据收集范围、使用目的、第三方共享、用户权利

---

### ❌ 雷区4：生成式AI内容未备案
如果产品有文字/图像/音视频生成能力，需进行AI生成内容备案

**解决方案：** 提前在工信部算法备案系统完成备案，获取备案号

---

### ❌ 雷区5：空壳智能体
只有介绍页面、没有实际功能 → 直接驳回

**解决方案：** 上架前至少完成1个完整的核心功能场景

## 六、上线后运营3个关键指标

智能体上线不是终点，以下数据要重点关注：

| 指标 | 健康值 | 优化方向 |
|:---|:---|:---|
| **对话完成率** | > 60% | 优化开场语和预置问题引导 |
| **工具调用成功率** | > 90% | 检查API稳定性，添加降级策略 |
| **用户复访率** | > 30% | 增加新内容/功能，保持新鲜感 |
