## 四、验收清单（签合同建议逐条写入）

| 验收维度 | 必验项 | 验收标准 |
| :--- | :--- | :--- |
| 功能完整性 | 合同功能清单逐条验收 | 通过率 100%，不允许“口头完成” |
| 性能 | 首页加载、下单、支付关键链路 | 高峰时段关键页面可用，无阻塞错误 |
| 稳定性 | 回归测试、异常处理 | 核心流程无 P0/P1 级缺陷 |
| 安全性 | 权限、数据、接口安全 | 管理后台权限隔离；敏感数据脱敏 |
| 可维护性 | 文档与交接 | 交付部署文档、接口文档、操作手册 |
| 可迁移性 | 资产归属 | 源码、数据库结构、账号权限完成交接 |

## 五、报价单必看“防坑条款”

1. 报价必须拆分到模块与子功能，不接受“打包总价”。
2. 明确“包含项/不包含项”（短信、支付通道费、云资源费等）。
3. 明确二次开发单价与评估时效（避免后续随意加价）。
4. 明确质保期、响应时效、修复时限（建议写入 SLA）。
5. 明确源码与知识产权归属、交付时间、交付形式。
6. 明确里程碑付款条件，不建议一次性预付。

## 七、使用建议

- 至少找 3 家供应商按同一模板报价。
- 同时看“首年总成本”和“次年总成本”。
- 单看低价没有意义，重点看可交付性和可维护性。
