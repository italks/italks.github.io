# AI 的"记性"为什么那么差？上下文窗口的真相

> 阅读时长：5分钟
>
> 20 万 Token 看起来很大，其实根本不够用。

---

## 你有没有遇到过这种情况？

和 AI 聊得好好的，突然它开始"失忆"：

```
你：还记得我刚才说的项目吗？
AI：不好意思，我没有看到您之前提到的项目信息。
```

或者：

```
你：帮我修改第 3 步的代码
AI：您之前提到了第 3 步，但我不太确定具体是什么内容。
```

**为什么 AI 会"失忆"？**

答案在两个字：**窗口**。

---

## 什么是上下文窗口？

**上下文窗口 = AI 的"大脑容量"。**

它决定了 AI 一次能"记住"多少内容。

### 类比一下

把 AI 想象成一个阅读者：

- **上下文窗口** = 一张白纸
- **对话内容** = 写在纸上的字

白纸大小有限，写满了就要擦掉旧的，才能写新的。

**这就是 AI"失忆"的原因：旧的对话被"擦掉"了。**

---

## 20 万 Token 到底能装多少？

Claude 的上下文窗口是 20 万 Token，看起来很大。

但换算一下：

| 内容 | Token 数量 |
|---|---|
| 1 个汉字 | 约 1-2 Token |
| 1 个英文单词 | 约 1 Token |
| 1 行代码 | 约 10-20 Token |

### 实际项目中的消耗

```
一个中等项目：
- 源代码：50 个文件 × 500 行 = 25000 行
- Token 消耗：25000 × 15 ≈ 37.5 万 Token
```

**结果：一个中等项目，代码就超过了 20 万 Token 限制。**

再加上对话历史、错误信息、API 文档……

**窗口很容易"爆"。**

---

## 为什么窗口会"爆"？

### 1. 代码量大

一个完整项目，代码量轻松超过 10 万 Token。

### 2. 对话历史累积

每次对话，都会消耗 Token：

```
你：帮我写个登录功能（10 Token）
AI：好的，我来实现登录功能...（500 Token）
你：再加个注册功能（10 Token）
AI：好的，我来添加注册功能...（500 Token）
...
```

对话越长，历史越长，Token 消耗越大。

### 3. 错误信息和调试日志

```
Error: Cannot read property 'map' of undefined
    at ProductList.tsx:42:15
    at renderWithHooks (react-dom.development.js:14985:18)
    at mountIndeterminateComponent (react-dom.development.js:17811:13)
    ...
```

错误信息、堆栈跟踪，都是 Token 消耗大户。

### 4. 文档和 API 说明

如果你粘贴了 API 文档、第三方库说明，Token 消耗会更快。

---

## 窗口爆了会发生什么？

### 1. AI 开始"失忆"

```
你：修改刚才的登录功能
AI：不好意思，我没有看到您之前提到的登录功能代码。
```

旧的对话被"擦掉"了，AI 不记得了。

### 2. 响应质量下降

AI 只能看到最近的对话，无法理解完整上下文：

```
你：用之前的技术栈
AI：您之前提到了技术栈，但我不确定具体是什么。
```

### 3. 甚至直接报错

```
Error: This conversation is too long. Please start a new conversation.
```

---

## 扩大窗口不是解药

有人说：把窗口扩大不就行了？

**问题在于：**

### 1. 成本

Token 越多，API 调用成本越高。

### 2. 速度

处理更多 Token，响应时间越长。

### 3. 效果

窗口越大，AI 越容易"迷失"在大量无关信息中。

**关键不是"装多少"，而是"装什么"。**

---

## 聪明的解法：不是扩大，而是管理

Claude Code 的做法是：**分层管理**。

### 三层记忆系统

| 层级 | 内容 | Token 消耗 |
|---|---|---|
| **MEMORY.md** | 轻量索引 | 始终在窗口中 |
| **CLAUDE.md** | 项目说明书 | 按需加载 |
| **会话上下文** | 当前对话 | 动态更新 |

**核心思想：**

- 不是把所有内容塞进窗口
- 而是按重要性分层管理
- 重要的常驻，次要的按需加载

---

## 类比：人类的记忆系统

AI 的三层记忆，类似人类的记忆：

| AI 记忆层 | 人类记忆 |
|---|---|
| MEMORY.md | 工作记忆（当前任务） |
| CLAUDE.md | 长期记忆（知识库） |
| 会话上下文 | 短期记忆（当前对话） |

**人类不会记住所有细节，而是记住索引，需要时查阅。**

AI 也应该这样。

---

## 对普通人的启发

### 1. 不要指望 AI 记住所有内容

AI 的"记性"有限，重要信息要主动重复。

### 2. 学会"分段"对话

长任务拆成多个短对话，避免窗口爆炸。

### 3. 善用文件而非对话

把重要信息写成文件（如 CLAUDE.md），让 AI 读取，而不是每次对话重复。

### 4. 定期"清理"对话

对话太长时，主动开启新对话，避免质量下降。

**AI 的"记性"差，是因为上下文窗口有限。**

- 20 万 Token 看起来大，实际项目很容易超过
