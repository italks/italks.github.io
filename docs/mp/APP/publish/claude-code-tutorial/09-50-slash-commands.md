## 一、代码管理类（8 个）

### 1. `/commit` —— 自动生成提交信息

**功能：** 分析当前变更，自动生成符合规范的提交信息并提交。

**用法：**

```
/commit
```

**示例输出：**

```
feat(search): add fuzzy search functionality

- Add SearchBar component with debounce
- Integrate search API with error handling
- Add unit tests for search logic
```

**适用场景：** 每次代码提交时。

---

### 2. `/review` —— 代码审查

**功能：** 分析当前变更，检查安全、性能、代码风格问题。

**用法：**

```
/review
```

**示例输出：**

```
## 代码审查结果

### 潜在问题
1. XSS 风险：用户输入未转义（Login.tsx:42）
2. 性能问题：循环中重复创建对象（utils.ts:18）

### 改进建议
- 添加错误边界处理
- 补充单元测试
```

**适用场景：** 提交前自查。

---

### 3. `/diff` —— 智能变更查看

**功能：** 用自然语言总结当前改动内容。

**用法：**

```
/diff
```

**示例输出：**

```
## 本次变更摘要

### 修改范围
- 文件数：3
- 新增行：45
- 删除行：12

### 主要改动
1. 新增搜索功能（SearchBar.tsx）
2. 修复 XSS 漏洞（sanitize.ts）
3. 补充单元测试（auth.test.ts）
```

**适用场景：** 提交前确认改动。

---

### 4. `/undo` —— 回滚最近操作

**功能：** 撤销 AI 最近的一次修改。

**用法：**

```
/undo
```

**适用场景：** AI 改错了，快速回滚。

---

### 5. `/redo` —— 重做操作

**功能：** 重新执行被撤销的操作。

**用法：**

```
/redo
```

**适用场景：** 误撤销后恢复。

---

### 6. `/history` —— 查看操作历史

**功能：** 列出最近的操作记录。

**用法：**

```
/history
```

**示例输出：**

```
## 最近操作

1. [10:30] 修改 App.tsx（添加路由）
2. [10:25] 创建 SearchBar.tsx
3. [10:20] 读取 CLAUDE.md
```

**适用场景：** 回顾操作记录。

---

### 7. `/branch` —— 分支管理

**功能：** 创建、切换、删除分支。

**用法：**

```
/branch create feature-search
/branch switch main
/branch delete old-feature
```

**适用场景：** Git 分支操作。

---

### 8. `/merge` —— 合并分支

**功能：** 合并当前分支到目标分支。

**用法：**

```
/merge main
```

**适用场景：** 功能开发完成后合并。
